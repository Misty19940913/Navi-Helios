# 35_電子報建設計畫

> [!important] 為什麼電子報是最重要的資產？
> IG / Threads / X 的追蹤者是「平台的資產」。演算法可以讓你的觸及率歸零。
> **電子報名單是你的資產**——它跟著你走，任何平台的算法改變都傷不到它。
> 所有社群內容的終極目標，只有一個：**把人導入電子報名單。**

---

## 一、策略定位

| 項目 | 內容 |
|:---|:---|
| **定位** | 轉化核心資產，而非單純內容渠道 |
| **發刊頻率** | 每週一封（固定週五或週日） |
| **內容方向** | 深度底層邏輯 / 案例分析 / 工具使用 |
| **CTA** | 每封信至少一個明確行動呼籲 |
| **目標名單** | 第一年：500 人 / 第二年：2,000 人 |

---

## 二、平台選擇

> [!important] 選台三大條件（依優先序）
> 1. **API 支援** → AI 可自動化操作
> 2. **中文內容支援** → 信件可用繁體中文撰寫
> 3. **免費額度** → 冷啟動期零成本

| 平台 | API | 中文信件 | 後台介面 | 免費額度 | 適用階段 |
|:---|:---:|:---:|:---:|:---|:---|
| **MailerLite** | ✅ | ✅ | 英文 | **1,000 人免費** | ✅ 冷啟動首選 |
| **Kit (ConvertKit)** | ✅ | ✅ | 英文 | **10,000 人免費** | ✅ 成長期考慮 |
| **Brevo** | ✅ 最完整 | ✅ | 英文為主 | 每日 300 封 | ✅ 規模化後 |
| **Substack** | ❌ | ✅ | 英文 | 免費 | ❌ 無 API，不選用 |

> [!note] 補充說明
> 所有主流平台均使用 UTF-8 編碼，**繁體中文信件內容完全支援**。
> 唯一差異是「後台操作介面」為英文——這不影響你的讀者收到中文信件。

**建議路線：**

```
冷啟動（0-1,000 人）： MailerLite  →  免費、API 完整、中文信件正常
                          ↓
成長期（1,000+ 人）：  Kit          →  創作者變現生態系最完整
                          ↓
規模化（10,000+ 人）： Brevo       →  多渠道 CRM + 複雜自動化
```

**AI 可以透過 API 幫你做的事：**
- 自動生成電子報草稿並排程發送
- 根據訂閱者行為（開信 / 點擊）自動分組
- 新訂閱者自動觸發歡迎序列
- 定期生成訂閱者數據報告


---

## 三、Lead Magnet 設計（獲取首批訂閱者）

電子報的第一個訂閱者，來自「給出足夠有價值的免費東西」。

**首選 Lead Magnet：**
> [[30_Projects/31_Planning/公司藍圖/03_產品/MVP-01_東黎現場勘查日記_開發企劃|《東黎：現場勘查日記》]]

- **形式**：PDF 指南 + Notion 模板
- **訂閱流程**：Threads/IG/X 貼文 → 留言 / 填表 → 自動發送 PDF → 進入電子報名單

---

## 四、執行時程

| 週次 | 任務 |
|:---|:---|
| **Week 1** | 申請 **MailerLite** 帳號、設定品牌頁面、獲取 API Key |
| **Week 2** | 完成 Lead Magnet（現場勘查日記 PDF） |
| **Week 3** | 發布第一篇「創刊號」，在 Threads/IG 公告電子報開放訂閱 |
| **Week 4** | 設計自動歡迎信（訂閱後立即發送） |
| **持續** | 每週固定發刊，在社群貼文中持續引導訂閱 |

---

## 五、衡量指標

| 指標 | 冷啟動目標（3個月） | 成長目標（1年） |
|:---|:---:|:---:|
| 訂閱人數 | 100 人 | 500 人 |
| 開信率 | > 40% | > 35% |
| 點擊率 | > 5% | > 8% |
| Lead Magnet 下載 | 50 次 | 200 次 |

---

## 六、電子報內容結構（每封信模板）

```
【主旨】：勾人的一句話（使用 Threads 測試過的高共鳴 Hook）

【開場】：一個讓人點頭的場景或問題（3-5 行）

【核心內容】：本週底層邏輯 / 案例分析 / 工具介紹（主體）

【洞察提煉】：用一句話總結本週核心觀點

【CTA】：明確的單一行動呼籲
  → 「回覆這封信告訴我你的情況」
  → 「點此下載本週工具」
  → 「轉發給一個需要這個觀點的人」
```

---

## 七、與社群渠道的分工

```
Threads / X（每日）
  → 測試觀點共鳴 → 高互動的觀點 → 擴寫成 IG 深度貼文 或 電子報主題
      
IG（每週）
  → 深度輪播圖 / 長文 → 結尾引導訂閱電子報

電子報（每週）
  → 深度版本 + 更個人的聲音 → CTA 指向付費產品
```
