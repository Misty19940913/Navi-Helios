# 最小工具集策略 (Minimalist Tech Stack)
> 來源：[[232-171數位工具箱與自動化流程：一人公司的鋼鐵裝]]

## 定義
一人公司的競爭力在於「消除摩擦」而非工具的複雜度。透過精選少數核心工具，降低維護成本。

## 工具矩陣
1. **內容與分銷**：Ghost / Substack / Beehiiv (擁有所有權)。
2. **生產力與大腦**：Obsidian / Notion (數位第二大腦)。
3. **自動化與連接**：Zapier / Make (數位連接器)。

## 機制
- **工具即摩擦**：每增加一個工具，就增加了一份複雜性與認知負荷。
- **最小化原則**：在業務尚未獲利前，拒絕昂貴的企業級軟體。
