# OKR模型
> [!abstract] 核心洞見 (Insight)
> OKR (Objectives and Key Results) 是一種目標管理框架，它將鼓舞人心的「目標」與可衡量的「關鍵結果」相結合，驅動個人與團隊專注於最重要的挑戰，實現突破性成長。

### 定義與機制 (What & How)
- **官方描述**：OKR 是一套嚴謹的思考框架和持續的紀律，旨在確保員工共同努力，並將精力聚焦於對組織有貢獻的可衡量事務上。它由一個定性的「目標 (Objective)」和數個定量的「關鍵結果 (Key Results)」組成，用於追蹤目標的達成進度。
- **我的理解**：OKR 就像是設定一場冒險遊戲。目標 (O) 是你要打敗的最終大魔王（例如：「成為全城最受歡迎的麵包店」），而關鍵結果 (KRs) 就是你打敗魔王前必須完成的幾個關卡（例如：「開發出 3 款明星商品」、「Google 評論達到 4.8 顆星」、「每月社群粉絲增長 20%」）。

- **運作原理**：OKR 透過設定一個有挑戰性的定性目標 (O)，並搭配 2-4 個可量化的關鍵結果 (KRs) 來運作。它採用如 [[PDCA]] 的短週期（通常為季度）循環，強調過程中的透明、協作與頻繁檢視，確保團隊方向一致並能快速調整。其核心是專注、對齊和激發潛能。
- **關鍵變數**：
    1.  **目標 (Objective)**: 必須是定性的、有啟發性的、有時間限制的。它回答「我們想去哪裡？」
    2.  **關鍵結果 (Key Result)**: 必須是定量的、可驗證的、有挑戰性的。它回答「我們如何知道正在前往目的地？」
    3.  **週期 (Cycle)**: 通常為季度，確保了敏捷性和適應性。
    4.  **透明度 (Transparency)**: 所有成員的 OKR 都是公開的，促進跨部門的對齊與協作。

### 實踐與案例 (Action & Proof)
- **具體案例**：Google 運用 OKR 實現了從瀏覽器到作業系統的多次重大創新。例如，一個提升 Chrome 瀏覽器性能的目標可能是：
    - **目標**：打造地球上最快的網頁瀏覽器。
    - **關鍵結果 1**：頁面載入時間減少 30%。
    - **關鍵結果 2**：渲染延遲降低至 60ms 以下。
    - **關鍵結果 3**：新分頁開啟速度提升 50%。
- **行動指南**：
    1.  **設定目標 (O)**：確定 1-3 個最重要、最具挑戰性的定性目標。
    2.  **制定關鍵結果 (KRs)**：為每個目標設定 2-4 個可衡量的、結果導向的指標。
    3.  **執行與追蹤**：在週期內定期更新進度（通常每週），保持團隊溝通。
    4.  **檢視與反思**：在週期結束時，為 KR 評分（通常為 0-1.0），分析成功與失敗原因，並為下個週期做準備。
- **情境關鍵字**：#目標設定, #團隊管理, #績效評估, #新創公司, #個人成長, #季度規劃

### 知識網絡 (Connection)
- **上層概念**：[[目標管理]], [[績效管理]]
- **對立概念**：KPI (Key Performance Indicator)。KPI 通常用於衡量「產出」，並與績效考核掛鉤，可能抑制挑戰精神；而 OKR 專注於「成果」，鼓勵設定挑戰性目標，通常與考核脫鉤。
- **衍生想法**：
    - OKR 的思想可以應用於 [[夢想人生金字塔]]，將長期夢想轉化為可執行的季度目標。
    - OKR 的迭代精神與 [[敏捷開發]] 的理念高度契合。
