# H3.0-問題解決流程 (Problem-Solving Process)

> [!abstract] 核心洞見 (Insight)
> 生活即解決問題；這不是程式漏洞 (Bug)，而是演化的核心功能。透過標準化的四步驟流程，能將痛苦的阻礙轉化為開發系統自主權（Agency）的課程。

### 定義與機制 (What & How)
- **核心定義**：一套將混亂的痛苦（Horror）轉化為結構化挑戰（Challenge）的操作手冊。
- **運作機制**：將問題從「情緒反應」中解耦，透過識別根源象限 -> 最小化方案 -> 連續性執行 -> 自動化整合，實現意識層級的實體化躍遷。

### 實踐與案例 (Action & Proof)
- **標準流程 (The 4-Step SOP)**：
  1. **識別 (Recognition)**：痛在哪裡？哪個象限是根源？解決後會創造什麼？是什麼在維持問題的存在？
  2. **設計 (Design)**：尋找最小有效劑量（MED）。設計每日/每週清單。設定真實的衡量標準與時間軸。
  3. **落地 (Implementation)**：從最小 viable 開始。先求「一致性（養成習慣）」，再求「複雜性（優化）」。調整反饋。
  4. **整合 (Integration)**：使其成為自動化習慣，連結到「我就是這樣的人」的身分認同。教導他人以鞏固知識。
- **思考協議**：面對問題時永遠要問：「這個解決方案能否增加我的反脆弱性？是否創造了與他人的正和博弈（Positive-sum）？」
