## 案例一：一人企業商業模式

### 1.1 案例背景

**角色**：Lily，32歲，前行銷經理  
**現況**：在科技公司工作5年，考慮成為自由工作者  
**目標**：建立一人企業，發展個人品牌

### 1.2 現況畫布（BMY-DRAW）

| 區塊 | 內容 |
|------|------|
| **客戶細分** | 科技公司行銷部門、中小企業主 |
| **價值主張** | 社群媒體營運、內容策略制定、行銷培訓 |
| **渠道通路** | LinkedIn、朋友推薦、接案平台 |
| **客戶關係** | 一對一服務、專案合作 |
| **收入來源** | 顧問費（$1500/專案）、培訓授課 |
| **關鍵資源** | 5年行銷經驗、大公司人脈、社群媒體技能 |
| **關鍵活動** | 客戶服務、內容創作、提案投標 |
| **合作夥伴** | 設計師、攝影師 |
| **成本結構** | 時間（每週50小時）、工具訂閱（$100/月） |

### 1.3 問題診斷

| 問題 | 影響 |
|------|------|
| 收入依賴單一來源 | 風險高，不穩定 |
| 時間換錢模式 | 收入天花板明顯 |
| 缺乏個人品牌 | 獲客依賴口碑 |

### 1.4 未來畫布（BMY-REVISE）

| 區塊 | 內容 |
|------|------|
| **客戶細分** | 專業人士（30-40歲）、個人品牌創作者、小型創業家 |
| **價值主張** | 個人品牌建立系統、內容創作方法論、職涯發展顧問 |
| **渠道通路** | 電子報（5000訂閱）、YouTube頻道、線上課程平台 |
| **客戶關係** | 社群經營、付費會員、一對一教練 |
| **收入來源** | 線上課程（$99-$499）、訂閱會員（$19/月）、教練服務（$200/小時）、企業培訓 |
| **關鍵資源** | 個人品牌成功案例、系統化方法論、內容創作能力、5000人網絡 |
| **關鍵活動** | 內容創作、課程開發、社群經營、個人成長 |
| **合作夥伴** | 課程平台、其他個人品牌、技術支援 |
| **成本結構** | 工具（$200/月）、課程製作（$500/門）、時間（每週30小時） |

### 1.5 轉型行動

| 行動 | 時程 | 目標 |
|------|------|------|
| 建立電子報 | 1-3月 | 1000訂閱 |
| 開發第一個課程 | 3-6月 | 上架銷售 |
| 建立付費會員 | 6-12月 | 100會員 |
| 減少顧問客戶 | 12月 | 全職內容創作 |

### 1.6 案例檢查清單

- [ ] 這個案例與我的情況相似嗎？
- [ ] 她的痛點跟我一樣嗎？
- [ ] 我可以複製她的哪些行動？
- [ ] 我的現況畫布長怎樣？
- [ ] 我的目標畫布長怎樣？

## 案例二：職涯轉型商業模式

### 2.1 案例背景

**角色**：Alex，28歲，軟體工程師  
**現況**：在金融公司工作3年，想轉行產品管理  
**目標**：成功轉職，並建立可持續的職業模式

### 2.2 現況畫布分析

| 區塊 | 現況 |
|------|------|
| **客戶細分** | 金融公司技術部門 |
| **價值主張** | 程式開發、技術解決 |
| **渠道通路** | 求職網站、獵頭 |
| **客戶關係** | 僱傭關係 |
| **收入來源** | 年薪80萬 |
| **關鍵資源** | Python、SQL、金融知識 |
| **關鍵活動** | 寫代碼、維護系統 |
| **合作夥伴** | 同事、主管 |
| **成本結構** | 時間、持續學習 |

### 2.3 轉型差距分析

| 差距 | 現在 → 未來 |
|------|--------------|
| **客戶細分** | 金融公司技術部門 → 科技公司產品團隊 |
| **價值主張** | 技術解決 → 技術+商業結合 |
| **收入來源** | 可能短期下降 |
| **關鍵資源** | 需要產品管理技能 |
| **關鍵活動** | 從寫代碼到策略決策 |

### 2.4 轉型行動方案

```
Phase 1: 準備（1-3月）
├── 學習產品管理知識
├── 完成相關認證
└── 建立產品作品集

Phase 2: 過渡（3-6月）
├── 內部轉崗機會
├── 兼職專案經驗
└── 擴展人脈網絡

Phase 3: 轉職（6-12月）
├── 申請產品經理職位
├── 談判薪酬
└── 入職新角色
```

### 2.5 風險管理

| 風險 | 應對策略 |
|------|----------|
| 收入下降 | 維持儲蓄6個月 |
| 技能不足 | 持續學習與練習 |
| 機會不足 | 多管道求職 |
| 心態調整 | 社群支持 |

### 2.6 案例檢查清單

- [ ] 我目前的客戶是誰？（誰付我錢）
- [ ] 我想轉到什麼類型的客戶？
- [ ] 我需要補足什麼技能？
- [ ] 我的轉型時程是多久？
- [ ] 我有足夠的儲蓄嗎？

## 案例三：產品服務商業模式

### 3.1 案例背景

**角色**：Sarah，35歲，在職媽媽  
**現況**：有教育碩士，想發展斜槓事業  
**目標**：建立線上教育事業

### 3.2 產品服務設計

| 產品層級 | 產品 | 價格 | 目標客群 |
|----------|------|------|----------|
| **免費** | 育兒電子書 | $0 | 新手媽媽 |
| **入門** | 線上課程 | $49 | 新手媽媽 |
| **進階** | 訓練營 | $199 | 有基礎媽媽 |
| **高端** | 一對一教練 | $999 | 高需求媽媽 |

### 3.3 完整商業模式

| 區塊 | 內容 |
|------|------|
| **客戶細分** | 在職媽媽、25-40歲、有大學學歷、注重自我成長 |
| **價值主張** | 工作家庭平衡方法、時間管理技巧、育兒信心建立 |
| **渠道通路** | Instagram（1萬粉絲）、電子報（2000訂閱）、媽媽社團 |
| **客戶關係** | 社群互動、私訊回覆、週報內容 |
| **收入來源** | 線上課程：30%、訓練營：40%、教練服務：20%、聯盟行銷：10% |
| **關鍵資源** | 教育碩士學位、3年育兒經驗、內容創作能力 |
| **關鍵活動** | 內容創作、課程開發、社群經營、客戶服務 |
| **合作夥伴** | 課程平台、母嬰品牌、其他媽媽KOL |
| **成本結構** | 工具：$100/月、行銷：$200/月、時間：每週15小時 |

### 3.4 驗證與迭代

| 假設 | 驗證方法 | 結果 |
|------|----------|------|
| 媽媽願意付費學育兒 | 預售測試 | 50人購買 |
| 線上課程有效 | 免費試聽 | 30%轉化 |
| 訓練營比課程好 | A/B測試 | 訓練營更受歡迎 |

### 3.5 案例檢查清單

- [ ] 我的產品矩陣是什麼？
- [ ] 每個產品的目標客群是誰？
- [ ] 我的定價邏輯是什麼？
- [ ] 我有驗證過假設嗎？
- [ ] 我的收入組合合理嗎？

## 案例四：從 0 到 1 的新創商業模式

### 4.1 案例背景

**產品**：時間追蹤 App  
**團隊**：3人聯合創辦人  
**階段**：早期驗證

### 4.2 商業模式設計

| 區塊 | 內容 |
|------|------|
| **客戶細分** | 自由工作者、小團隊（<10人）、遠端工作者 |
| **價值主張** | 自動時間追蹤、專案成本分析、發票生成 |
| **渠道通路** | 產品 Hunt、Indie Hacker、SEO內容 |
| **客戶關係** | 產品內引導、電子郵件支持、文檔教程 |
| **收入來源** | 訂閱（$9-29/月）、企業版（$99/月） |
| **關鍵資源** | 技術開發能力、使用者洞察、快速迭代 |
| **關鍵活動** | 產品開發、用戶獲取、功能優化 |
| **合作夥伴** | 雲端儲存、發票系統、支付閘道 |
| **成本結構** | 開發人力、伺服器、行銷 |

### 4.3 迭代歷程

```
Version 1.0（付費牆）
├── 發現：使用率低
└── 迭代：改為 Freemium

Version 2.0（只有追蹤）
├── 發現：功能太單一
└── 迭代：增加報告功能

Version 3.0（專業用戶）
├── 發現：付費用戶是大團隊
└── 迭代：專注企業市場
```

### 4.4 案例檢查清單

- [ ] 我用什麼方法驗證？
- [ ] 驗證失敗了怎麼辦？

## 案例學習總結

### 共同模式

1. **從現況到未來**：所有案例都從現況畫布開始
2. **差距驅動**：差距分析指引行動方向
3. **迭代驗證**：透過MVP測試假設
4. **逐步過渡**：不期望一次成功

### 關鍵成功因素

| 因素 | 說明 |
|------|------|
| **市場契合** | 產品符合市場需求 |
| **差異化** | 有明確的獨特之處 |
| **可持續性** | 收入大於成本 |
| **迭代能力** | 能快速回應反饋 |

### 應用流程

```
Step 1: 選擇案例
   └─ 找最接近你情況的案例
           ↓
Step 2: 畫現況
   └─ 用九宮格描繪現在狀態
           ↓
Step 3: 找差距
   └─ 現況與目標的差距
           ↓
Step 4: 定行動
   └─ 規劃具體的转型行動
           ↓
Step 5: 驗證
   └─ 用最小成本測試
```
