# 030_價值主張設計

> 本章節探討價值主張的核心概念、設計方法，以及如何從價值主張過渡到實際產品與服務。
>
> **版本**：1.0.0 | **更新**：2026-04-27

---

## 一、為何價值主張是商業模式的核心？

價值主張回答一個問題：**「為什麼客戶該選擇你，而不是競爭者？」**

多數人在這裡卡住——他們有技能、有熱情，但不知道如何把「會做的事」包裝成「客戶願意付費的價值」。

> 技能不等於價值。價值是技能在客戶身上產生的**可見改變**。

---

## 二、痛點對應法：從客戶的傷口出發

### 核心流程

Wiki 知識：`[[K-BIZ-013_痛點對應法]]`

```
痛點清單 → 解決方案 → 價值主張
```

從目標客戶的**真實痛點**出發，反向設計解決方案與價值主張。

### 痛點三層次

| 層次 | 說明 | 範例 |
|------|------|------|
| **淺層症狀** | 客戶自己描述的問題 | 「我時間不夠用」 |
| **深層原因** | 問題背後的根本原因 | 「因為不會分配優先級」 |
| **根本恐懼** | 如果不解決會怎樣 | 「40歲還在基層」 |

> 層次越深，價值主張越有力。

### 傷痕分析法

Wiki 知識：`[[K-BIZ-015_傷痕分析法]]`

> **痛點即市場。你的痛苦在哪裡，你的天賦就在哪裡。**

步驟：
1. 回顧自己經歷過的重大痛點
2. 分析痛點背後的技能與知識
3. 將解決自己問題的方法產品化

## 三、價值主張的12種元素

Wiki 知識：`[[K-BIZ-002_價值主張元素]]`

### 量化 vs 質化導向

| 導向 | 核心 | 範例 |
|------|------|------|
| **Quantitative** | 價格與效率 | 省時50%、省錢30% |
| **Qualitative** | 客戶體驗與成果 | 內心平靜、自信提升 |

### 12種提供元素

| 元素 | 說明 | 範例 |
|------|------|------|
| **Newness** | 創新全新的解決方案 | iPhone |
| **Performance** | 提升產品/服務性能 | 高效能筆電 |
| **Customization** | 客製化滿足個別需求 | 訂製西裝 |
| **Getting the job done** | 完成任務導向 | 代駕服務 |
| **Design** | 設計美學導向 | 蘋果設計 |
| **Brand/Status** | 品牌與地位象徵 | Louis Vuitton |
| **Price** | 價格競爭力 | 廉價航空 |
| **Cost reduction** | 降低成本 | SaaS 取代軟體授權 |
| **Risk reduction** | 降低客戶風險 | 保固服務 |
| **Accessibility** | 增加可及性 | 共享單車 |
| **Convenience/Usability** | 便利性與易用性 | 自動販賣機 |

## 四、差異化定位：讓你無法被複製

Wiki 知識：`[[K-BIZ-014_差異化定位法]]`

### 三維度交叉定位

```
          專業深度
             ×
        方法論
             ×
        經驗背景
             ↓
        獨一無二定位
```

| 維度 | 關鍵問題 |
|------|----------|
| **專業深度** | 我的專業領域有何獨特之處？ |
| **方法論** | 我的解決方法有何不同？ |
| **經驗背景** | 我的人生經歷如何獨一無二？ |

### 定位公式

Wiki 知識：`[[K-BIZ-011_定位公式]]`

```
(本質工作 × 業餘專長) + 生命痛點 = 唯一性標籤
```

**範例**：
- 本質工作：行銷經理
- 業餘專長：業餘畫畫
- 生命痛點：曾經創業失敗
- 定位：幫助創業者用視覺化行銷避免失敗的行銷教練

## 五、從價值主張到產品矩陣

Wiki 知識：`[[K-BIZ-016_產品矩陣]]`

### 一人企業的產品階梯

```
免費產品 → 入門產品 → 進階產品 → 高端產品
  (流量)     (轉化)      (營收)      (深度)
```

|| 層級 | 功能 | 目標 |
|--|------|------|------|
| **免費產品** | 吸引流量、建立信任 | 擴大受眾 |
| **入門產品** | 低門檻轉化 | 初次購買 |
| **進階產品** | 主要營收來源 | 深度服務 |
| **高端產品** | 高價值服務 | 頂級客戶 |

### 驗證與規模化路徑

Wiki 知識：`[[K-BIZ-020_變現階梯]]`

```
Stage 1: 實證服務 (PoC)
   → 用技能換取現金，收集痛點與驗證方法論

Stage 2: 經驗模式化
   → 將解決方案流程化、模板化

Stage 3: 百萬美元產品
   → 實現時間槓桿，產品在你睡覺時工作
```

## 六、應用案例

### Sarah 的價值主張設計

**背景**：在職媽媽，有教育碩士學位，想建立線上教育事業。

**Step 1：找痛點**
- 親身經歷：時間管理困難、育兒信心不足
- 傷痕分析法：自己走過的路最懂

**Step 2：差異化定位**
- 專業深度：教育碩士
- 方法論：正向教養 + 職場媽媽實戰經驗
- 經驗背景：親身驗證過的路

**Step 3：產品矩陣**
- 免費：育兒電子書（時間管理技巧清單）
- 入門：線上課程 $49（新手爸媽必修課）
- 進階：訓練營 $199（實作工作坊）
- 高端：一對一教練 $999（個別指導）

**驗證**：預售測試 → 50人購買 → 30%轉化到進階

## 七、檢查清單

### 設計前檢查

- [ ] 這個痛點是真實存在的嗎？還是我想像的？
- [ ] 我自己經歷過這個痛點嗎？（傷痕分析法：[[K-BIZ-015_傷痕分析法]]）

### 設計中檢查

- [ ] 價值主張使用了哪些元素？（12種元素的哪幾種？）
- [ ] 是量化導向還是質化導向？
- [ ] 三維度定位的交叉點是什麼？
- [ ] 定位公式算出來的標籤是什麼？

### 完成後檢查

- [ ] 客戶看完會說「對！這就是我需要的」嗎？
- [ ] 跟競爭者的差異清楚嗎？
- [ ] 產品矩陣有層次嗎？（免費 → 高端）
- [ ] 有沒有從服務升級到產品的路徑？（變現階梯）

---

## 本章小結

- 價值主張從**痛點**出發，不是從技能出發
- 12種元素幫你系統化盤點價值組合
- 三維度差異化確保無法被複製
- 產品矩陣是價值主張的具體化呈現
- 變現階梯是從服務到產品的進化路徑
