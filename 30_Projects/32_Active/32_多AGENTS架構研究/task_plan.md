## 組織架構

### L1：規劃層

| 角色 | 數量 | 功能 |
|------|------|------|
| 太子 | 3（目前僅 taizi）| 各自分管一個專案，接收旨意、規劃執行方案 |

### L2：協調層（雙向把關）

| 方向 | 職責 | 對應功能 |
|------|------|---------|
| 上而下 | 審查 L1 計劃、發佈任務 | menxia（審議） |
| 下而上 | 檢查 L3 成果、上報/駁回 | xingbu（質量保障） |

**皇上最終確認環節：**
- 計劃確認：L2 審查通過後，皇上確認是否發佈
- 成果確認：L2 檢查通過後，皇上確認是否上報

### L3：執行層

| Agent | 專業領域 | 任務屬性 |
|-------|---------|---------|
| hubu | 數據分析、統計、資源管理、報表生成 | finance, data |
| libu | 文檔、規範、用戶界面、對外溝通 | writing, ui |
| libu_hr | 人事管理、團隊建設、Agent 管理、培訓 | hr |
| bingbu | 功能開發、架構設計、重構優化 | code, architecture |
| gongbu | 基礎設施、部署運維、性能監控 | devops |
| xingbu | 質量保障、測試驗收、程式碼審查 | qa |
| doc-analyster | 分析研究 | research |
| doc-architecter | 架構設計 | architecture |
| doc-editor | 編輯校對 | writing |

## 任務流向（六環）

| 環節 | 角色 | 操作 |
|------|------|------|
| 1. 規劃 | L1（太子A/B/C）| 各自規劃專案 |
| 2. L2 審查 | menxia | 可行性、完整性審查 |
| 3. 皇上確認 | 你（navi）| 最終確認後發佈 |
| 4. 發佈任務 | Task Queue | L3 輪詢領取 |
| 5. L3 執行 | 六部、執行 Agent | 主動輪詢領任務、執行 |
| 6. L2 檢查 | xingbu | 質量審查 |
| 7. 皇上確認 | 你（navi）| 最終確認後上報 |
| 8. 回奏 | taizi | 彙總呈報皇上 |

---

## 任務派發方式

| 情境 | 方式 |
|------|------|
| 皇上直接下旨 | taizi 分揀 → 三省六部流程 → 皇上確認 → 發佈 |
| 日常例行任務 | L2 直接發佈到 Task Queue，L3 輪詢領取 |
| 多專案並行 | 三個太子各自管理，L2 統一協調 |

---

## 吸收的機制

| 機制 | 來源 | 應用 |
|------|------|------|
| 任務流向（上而下/下而上）| 三省六部 | L1→L2→L3 / L3→L2→L1 |
| 審查機制 | 三省六部 | menxia 審查計劃、xingbu 審查成果 |
| 任務輪詢 | Mission Control | L3 Agent 主動到 Queue 領取任務 |
| 皇上最終把關 | 原生設計 | 計劃確認、成果確認 |
