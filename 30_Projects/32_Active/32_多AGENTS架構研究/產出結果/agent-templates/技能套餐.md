# 技能套餐配置

## 核心模板檔案

建立新 Agent 時，從以下模板複製：

| 檔案 | 說明 | 來源 |
|------|------|------|
| AGENT.md | 行為規範、工作職責、工具權限 | agent-templates/ |
| SOUL.md | 靈魂、人格、價值觀 | agent-templates/ |
| USER.md | 用戶資訊、偏好 | agent-templates/ |
| IDENTITY.md | 身份識別、名稱、emoji | agent-templates/ |
| TOOLS.md | 本地工具備忘錄 | agent-templates/ |

---

## 共享技能（自動繼承）

這些技能會自動提供給所有 Agent：

| 技能 | 功能 | 來源 |
|------|------|------|
| defuddle | 從網頁提取乾淨內容 | ~/.openclaw/skills/defuddle |
| json-canvas | 編輯 JSON Canvas 檔案 | ~/.openclaw/skills/json-canvas |
| obsidian | Obsidian vault 管理 | ~/.openclaw/skills/obsidian |
| obsidian-bases | Obsidian Bases 資料庫 | ~/.openclaw/skills/obsidian-bases |
| obsidian-cli | Obsidian CLI 操作 | ~/.openclaw/skills/obsidian-cli |
| obsidian-markdown | Obsidian Markdown 語法 | ~/.openclaw/skills/obsidian-markdown |
| planning-with-files | 檔案式計畫管理 | ~/.openclaw/skills/planning-with-files |

---

## 特殊技能（需要額外配置）

這些技能需要特別設定，不會自動繼承：

| 技能 | 功能 | 配置方式 |
|------|------|----------|
| self-evolve | 自我進化協議 | 僅 L1 主管層級 |
| self-improving-agent | 錯誤學習記錄 | 全域可用 |
| skill-vetter | 技能安全審查 | 安裝新技能時使用 |

---

## Per-Agent 技能配置

每個部門可以有自己的專屬技能。

請參考 `AGENT.md` 中的「工具權限配置」章節來設定。

### 技術部門 (TechLead)
```json5
{
  "skills": {
    "inherited": ["defuddle", "json-canvas", "obsidian-*", "planning-with-files"],
    "dedicated": ["github", "gog"]
  }
}
```

### 媒體部門 (MediaLead)
```json5
{
  "skills": {
    "inherited": ["defuddle", "json-canvas", "obsidian-*", "planning-with-files"],
    "dedicated": []
  }
}
```

### 文書部門 (DocLead)
```json5
{
  "skills": {
    "inherited": ["defuddle", "json-canvas", "obsidian-*", "planning-with-files"],
    "dedicated": []
  }
}
```

## 安裝新技能

當需要安裝新技能時：

1. **使用 skill-vetter 審查**
   ```bash
   clawhub search <技能名稱>
   ```

2. **驗證安全後安裝**
   ```bash
   clawhub install <技能名稱>
   ```

3. **配置到特定 Agent**
   - 在該 Agent 的 workspace 建立 `skills/` 目錄
   - 或在 openclaw.json 中配置

---

## 技能繼承說明

- **全域技能**：所有 Agent 自動可用
- **部門技能**：在部門 workspace 的 `skills/` 目錄
- **個人技能**：僅該 Agent 可用
