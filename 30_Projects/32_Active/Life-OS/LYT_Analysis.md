## 一、LYT Kit 是什麼？

**LYT（Linking Your Thinking）** 是 Nick Milo 開發的個人知識管理（PKM）方法論，核心主張：

> **筆記不是用來「存放」的，是用來「生長」的。**

LYT 的設計哲學：讓結構從使用中**有機浮現**，而非由上而下強制套用。

---

## 二、LYT 的哲學基礎

### 2.1 Note-making vs. Note-taking

| 類型 | 行為模式 | 長期結果 |
|------|----------|----------|
| **Note-taker**（記錄者）| 抄下資訊，短期使用後拋棄 | 「燒完即棄」—— 知識不累積 |
| **Note-maker**（造筆者）| 主動生成洞察，長期耕耘 | 「知識資產」—— 越用越有價值 |

**LYT 的目標**：讓使用者成為 Note-maker。

### 2.2 心理擠壓點（Mental Squeeze Point）

> 當未整理的資訊量大到讓你感到不知所措時，這就是「Mental Squeeze Point」。

這是創建 MOC（內容地圖）的**有機觸發點**，不是事先規劃的，而是自然發生的需求。

### 2.3 Fluid Frameworks（流動框架）

- 結構是**流動的**，不是剛性的
- 筆記可以同時存在於多個脈絡中（**非破壞性思考**）
- 允許**Heterarchy**（異位階制）：同一個元素可以在多種排序方式下被找到

## 三、核心框架：MOC（Map of Content）

### 3.1 MOC 是什麼？

**MOC（內容地圖）** 是一種**高階筆記**，用來：
- 聚集（Gather）相關筆記
- 發展（Develop/Collide）筆記之間的關係
- 導航（Navigate）進入某個主題的入口

> MOC 不是目錄，是「**工作台**」——一個讓想法碰撞的空間。

### 3.2 MOC 的三個成長階段

```
Phase 1：Gather（聚集）
└── 把相關筆記都拉到同一個地方
    └── 此時 MOC 是「工作桌」

Phase 2：Collide（碰撞/發展）
└── 讓想法互動，找關係，生成新洞察
    └── 此時 MOC 是「熔爐」

Phase 3：Navigate（導航）
└── MOC 成熟後成為該主題的導航中心
    └── 此時 MOC 是「羅盤」
```

**範例演示（Habits MOC）：**
- **Gather**：把所有習慣相關筆記全部列在一起
- **Collide**：開始分群、找關聯，例如「習慣迴圈」vs「觸發機制」
- **Navigate**：最終形成有架構的習慣知識地圖

### 3.3 MOC 的類型

| 類型 | 說明 |
|------|------|
| **Home Note** | 整個 vault 的「首頁」，最高層次的 MOC |
| **Category MOC** | 某個主題/領域的匯聚地圖（如 Habits MOC） |
| **Timeline MOC** | 時間軸結構的 MOC |

## 四、ACCESS 資料夾系統

LYT 推薦一套 7 個資料夾的 Vault 組織架構：

```
ACCESS
├── Encounters   → 臨時捕捉區（未整理的新想法/輸入）
├── Atlas        → 地圖區（Home Note + 所有高階 MOC）
├── Calendar     → 時間戳記筆記（日誌/會議記錄/運動記錄）
├── Cards        → 知識卡片（原子知識單元/常青筆記）
├── Extras       → 工具箱（圖片/附件/模板）
├── Sources      → 外部資料來源（書/Podcast/文章——別人說的話）
└── Spaces       → 生活領域分區（工作/個人/興趣，若需隔離）
```

### 各資料夾詳解

**Encounters（相遇）**
- 靈感捕捉的「收件匣」
- 未整理、未分類的快速記錄
- 類似 GTD 的 Inbox 概念

**Atlas（地圖集）**
- 「地圖的地圖」
- 包含 Home Note 和所有頂層 MOC
- 這是思考的**主要工作空間**

**Calendar（日曆）**
- 所有有時間屬性的筆記
- 日記、週記、會議記錄
- 不是「重要概念」，是「時間切片」

**Cards（卡片）**
- 原子級知識單元
- Evergreen Notes 住在這裡
- 每張卡片是一個完整的概念

**Extras（附件）**
- 圖片、PDF、CSS 等支援材料
- 模板也放在這裡

**Sources（來源）**
- 別人說的話、讀書筆記、文章摘要
- 與 Cards（你自己消化的）分開

**Spaces（空間）**
- 生活的「大區塊」分區
- 例如 Work、Personal、Hobby
- 適合需要分離不同生活面向的人

## 五、Evergreen Notes（常青筆記）

### 5.1 定義

> 常青筆記 = 隨時間持續演化的活筆記

### 5.2 三大特徵

| 特徵 | 說明 |
|------|------|
| **原子性（Atomic）** | 一張筆記只包含一個核心概念 |
| **用自己的語言寫** | 不是轉貼，是消化後重組 |
| **高度連結** | 透過連結和其他筆記建立關係 |

### 5.3 Evergreen vs. 普通筆記

```
普通筆記：「2026-04-14 讀書筆記」
→ 某個時間點的快照，不再更新

Evergreen Note：「習慣的形成需要情境設計」
→ 每次新知識都回來補充、修正
→ 越來越豐富的知識資產
```

## 六、Home Note

### 6.1 什麼是 Home Note？

Home Note = Vault 的**起點**和**控制中心**

- 它是一個 MOC，組織的是「努力方向（Efforts）」
- 不是列出所有筆記，而是列出**最重要的進入點**

### 6.2 Home Note 的結構

Nick Milo 建議用「**Efforts**」而非「Projects」組織 Home Note：

```
Efforts（努力方向）
├── Ongoing（持續進行中）
├── Starting（剛啟動）
├── Paused（暫停中）
└── Completed/Abandoned（完成/放棄）
```

**為什麼用 Efforts 而不是 Projects？**
- Projects 有開始/結束，太過正式
- Efforts 更靈活，涵蓋「習慣」「興趣」「長期項目」

### 6.3 Home MOCs（首頁 MOC 群）

Home Note 連結到各個「首頁 MOC」：

```
Home Note
├── Thinking（思考）MOC
├── Health（健康）MOC
├── Finance（財務）MOC
├── Relationships（關係）MOC
└── 自定義...
```

| LYT 概念 | Life-OS 現有對應 | 相容性 |
|---------|----------------|--------|
| Home Note | Life-OS-MOC.md | ✅ 高度相容 |
| Knowledge | 20_Life_Planning/ | ✅ 基本相符 |
| Calendar | 21_Journsal/ | ✅ 完全相符 |
| Cards | 待定（無 Cards 資料夾）| ⚠️ 缺少 |
| Encounters | 無 Inbox 概念 | ⚠️ 缺少 |
| Extras | 80-System/ | ✅ 功能相符 |
| Sources | 無 Sources 資料夾 | ⚠️ 缺少 |
| Spaces | 30_Projects（部分）| 🟡 部分相符 |
| MOC（三階段）| Goal/Area/Project MOC | ✅ 邏輯相符 |
| Evergreen Notes | 10_Cards/（原子頁）| ✅ 已有機制 |
| Efforts（Home 結構）| Goal 三層系統 | 🟡 可整合 |

### 8.2 Life-OS 目前最大的缺口（對比 LYT）

**缺口 1：無 Inbox/Encounters 機制**
- LYT 的 Encounters 是系統能持續運作的關鍵入口
- 你的 Life-OS 目前沒有「快速捕捉 → 整理」的標準流程

**缺口 2：Cards 與 MOC 的連動不清楚**
- LYT 裡，Cards（原子知識）是 MOC 的「原料」
- Life-OS 有 10_Cards，但 Cards → Area/Goal MOC 的流向不明確

**缺口 3：Sources 分離**
- LYT 明確區分「別人說的（Sources）」vs「我自己消化的（Cards）」
- 這個分離在你的 vault 中尚不清楚

### 8.3 LYT 對 Life-OS 的啟發

1. **Efforts 概念可補充 Goal 系統**：Goal 是正式的戰略層，Efforts 可補「持續進行中的事」這個模糊地帶
2. **Mental Squeeze Point 觸發 MOC**：Life-OS 的 Area MOC 應該是「有機成長的」，不是一次設計好就不動
3. **三階段 MOC 流程**：Gather → Collide → Navigate 可以成為 Life-OS 的「MOC 建立 SOP」

### 10.2 重要新發現：Efforts 的實際運作機制

#### Efforts vs Projects 的根本差異

| 維度 | Projects | Efforts |
|------|----------|---------|
| 性質 | 正式，有截止日期 | 流動，可持續或暫停 |
| 心理壓力 | 高（deadline 驅動）| 低（給想法呼吸空間）|
| 適用類型 | 明確交付物 | 習慣、興趣、長期探索 |
| 觸發機制 | 外部要求 | 內在驅動 |

**Nick 的核心主張**：
> 「Projects 有截止日期，會扼殺創意。Efforts 提供語言上的緩衝空間，讓想法有地方呼吸。」

#### Efforts 的 Dataview 追蹤

任何筆記只要標記 `#effort` 標籤，就會自動出現在 Efforts 追蹤清單：

```dataview
LIST
FROM #effort
WHERE status = "ongoing"
```

### 10.3 Inbox 的「冷卻墊（Cooling Pad）」機制

**概念**：Inbox 不是「待辦清單」，而是讓想法「冷卻」的地方。

```
想法進來（熱的、衝動的、混亂的）
    ↓ 放入 Encounters（+ 資料夾）
冷卻幾天（不要立刻處理）
    ↓ 定期整理時再來
評估：值得深化？還是刪除？
    ↓
進入 Notebox（發展中）或 Archive（封存）
```

**關鍵洞察**：不強迫自己立刻處理，避免 Inbox 變成壓力來源。

### 10.4 Notebox 的筆記演化狀態

| 狀態 | 標籤 | 說明 |
|------|------|------|
| **Boats** 🛥️ | `#note/boat` | 孤立的快速記錄，尚未連結 |
| **Develop** 🌱 | `#note/develop` | 值得深化和連結的筆記 |
| **Evergreen** ❄️ | 無特定標籤 | 已成熟，持續演化的知識卡片 |

### 10.5 Plan and Review 的「年度主題」系統

Nick 不用「年度目標」，而是用 **「年度主題（Annual Theme）」**：

- 主題比目標更靈活，不會因為沒達成而失敗
- 例：2022 主題 → 「筆記製作者的歸來（The Note-maker's Return）」
- 主題引領整年的覆盤方向

覆盤結構：
```
年度覆盤
├── 反思（Reflections）— 這一年發生了什麼？
└── 作戰計畫（Game Plan）— 下一年的行動方向
```

### 10.6 Library 的 LYT 分類系統

Library 採用改良版的 **000-900 分類法**（類似杜威十進制）：

```
000 - 知識與系統（Meta）
100 - 哲學與思考
200 - 心理與行為
300 - 社會與文化
400 - 語言與溝通
500 - 科學與自然
600 - 技術與工程
700 - 藝術與創造
800 - 文學與敘事
900 - 歷史與地理
```

**設計意圖**：讓人類的所有知識都有「家」，避免筆記無處可歸。

### 10.7 Outbox：輸出管理

## 十四、實體 Vault 分析：Ideaverse Lite 1.5

> **路徑**：`c:\Users\安泰\OneDrive\Obsidian\Ideaverse Lite 1.5`
> **定位**：LYT 官方提供的實踐範本，展示了系統簡化後的 **ACE 框架**。

### 14.1 ACE 資料夾框架（ACE Folder Framework）

Ideaverse Lite 將複雜的 ACCESS 系統精簡為三個核心資料夾：

1. **A - Atlas（地圖集）**
   - `Maps/`：存放所有 MOC（如 Thinking Map, Library）。
   - `Dots/`：存放原子筆記，進一步細分為 `People`, `Sources`, `Statements`, `Things`。
   - **關鍵發現**：`Statements` 資料夾是 Evergreen Notes 的所在地（筆記標題即觀點）。

2. **C - Calendar（日曆）**
   - 存放所有時間相關筆記。
   - 包含 `Logs/`（日誌）和 `Notes/`（週期性筆記）。

3. **E - Efforts（努力方向）**
   - 取代傳統的 "Projects"。
   - 依**強度（Intensity）**分類：
     - `On`：當前核心任務（火熱）。
     - `Ongoing`：持續進行中。
     - `Simmering`：慢火燉煮（低優先級）。
     - `Sleeping`：休眠中。

### 14.2 核心 MOC：Library 的分類體系

Ideaverse 使用了改良版的杜威十進位法（000-900）來組織 `Atlas`：

- **000 - Knowledge Management**（PKM 工具與方法）
- **100 - Personal Management**（身心健康、習慣、人生地圖）
- **200 - Philosophy & Psychology**
- **300 - Social Sciences**（人際關係、社會）
- **400 - Communications & Language**
- **500 - Natural Sciences**
- **600 - Applied Sciences**（技術、AI、工程）
- **700 - Art & Recreation**（電影、音樂、運動）
- **800 - Literature**
- **900 - History & Geography**

### 14.3 實作細節：YAML 與連結語法

- **向上連結（Upward Link）**：每份筆記都有 `up: [[Parent MOC]]`，確保底部向上（Bottom-up）的導航。
- **MOC 模板**：使用大量 **Callouts**（如 `[!Map]`, `[!Box]`）來建立美觀且模組化的視覺介面。
- **Efforts 追蹤**：首頁使用 Dataview 語法從 `Efforts/On` 等資料夾自動抓取清單。
