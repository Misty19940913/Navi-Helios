# 01_核心參數

## 兩種資金定義（最重要）

### 風險基準 N₀（靜態錨點）

- **用途**：計算最大名義部位、各層上限、極端情境檢核
- **更新時機**：
  - 明確出金
  - 明確入金
  - ❌ 不因浮虧浮盈改變
- **當前設定**：1000 USDT

### 操作資金 N₁（動態浮動）

- **用途**：計算單筆建倉、補倉金額
- **特性**：隨帳面實際資金自動變動
- **連動**：資金增加 → 單筆放大；資金減少 → 單筆縮小

---

## 槓桿與名義上限

### 理論最大槓桿

```
最大槓桿 = 3×
```

### 總名義上限

```
Max_Notional_Total = N₀ × 3 = 3000 USDT
```

---

## ADI 動態縮放係數 E

### 公式

```
E = 0.6 + 0.4 × (ADI / 100)
```

### 行為對照

| ADI 值 | E（縮放係數） | 市場狀態 | 可用名義部位 |
|--------|--------------|----------|--------------|
| 0 | 0.6 | 橫盤/無趨勢 | 保守模式 |
| 25 | 0.7 | 弱趨勢 | 適度放大 |
| 50 | 0.8 | 中等趨勢 | 正常放大 |
| 75 | 0.9 | 強趨勢 | 接近上限 |
| 100 | 1.0 | 極強趨勢 | 完全放大 |

### 核心邏輯

- ADI 高 → 牛市/明顯趨勢 → E ↑ → 可放大名義部位
- ADI 低 → 橫盤無方向 → E = 0.6 → 保守操作

---

## 單筆建倉與補倉

### 建倉（首次入場）

```
Entry_Size = N₁ × 10% × E
```

### 補倉（加碼）

```
Add_Size = N₁ × 5% × E
```

### 觸發條件

- 價格回撤 ≥ 1 ATR
- 且滿足 L/C/T 各層行為規則

---

## 關鍵約束

> ⚠️ 單筆建倉/補倉金額 **永遠不能突破** 各層 / 各幣的有名義上限

---

## 實例計算

假設：
- N₀ = 1000
- N₁ = 1000
- ADI = 50 → E = 0.8
- L 層 BTC，佔 L 的 40%

### Step 1：總名義上限

```
Max = 1000 × 3 × 0.8 = 2400 USDT
```

### Step 2：L 層上限

```
L_max = 2400 × 50% = 1200 USDT
```

### Step 3：BTC 上限

```
BTC_max = 1200 × 40% = 480 USDT
```

### Step 4：單筆建倉

```
Entry = 1000 × 10% × 0.8 = 80 USDT
```

### Step 5：單筆補倉

```
Add = 1000 × 5% × 0.8 = 40 USDT
```

---

## 鐵律

1. **N₀ 只在明確出入金時更新**
2. **單筆金額由 N₁ 決定，但上限由 N₀ × E 控制**
3. **E 由 ADI 決定，趨勢越強放大越多**
