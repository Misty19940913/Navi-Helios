# 05_極端情境與末日假設

## 核心理念

> **在黑天鵝發生之前，就已經決定好如何應對。**

---

## 末日假設定義

### 單日極端跌幅

```
Max_Daily_Drop = -40%
```

### 為什麼是 -40%？

- 2020 年 3 月 13 日：BTC 日內最大跌幅 ≈ 40%
- 2022 年 11 月 FTX 崩潰：主流幣單日跌幅 25–35%
- 加密市場結構性高相關：多幣同跌是常態，非例外

### 前提

> 所有標的高度正相關（BTC 領導市場）
> 因此 **不是單一幣的 -40%，是整個帳戶的 -40%**

---

## 末日假設下的數學

### 可承受虧損計算

```
最大可承受虧損 = N₀ × -40% = -400 USDT
```

### 實際最大名義部位

```
Max_Notional = N₀ / 40% = 2500 USDT（理論）
Max_Notional_With_E = 2500 × E
```

### 為何設定 3× 而不是 2.5×？

```
3× → 可承受 33% 虧損
-40% → 可承受 40% 虧損

存在安全差距 ≈ 7%（約 70 USDT）
```

---

## 全倉同步縮放機制

### 為什麼要同步縮放？

當帳面資金減少時：

```
N₁ 減少 → 單筆建倉/補倉金額減少
       → 所有層級同時變小
       → 不會被迫在虧損期放大槓桿
```

### 縮放公式

```
實際層級上限 = 理論層級上限 × (N₁ / N₀)
```

### 實例

- N₀ = 1000
- N₁ 下跌至 800
- L 層理論上限 = 1500

```
L_實際 = 1500 × (800 / 1000) = 1200 USDT
```

## 極端情境行為腳本

### 情境 1：單日 -40% 暴跌

**觸發條件**：任意主流幣單日跌幅 ≥ 40%

**系統反應**：

1. 所有 T 層標的 → **預設歸零，不補**
2. C 層標的 → **停止補倉，評估是否止損**
3. L 層標的 → **只評估是否在結構容忍範圍內**
4. 禁止新建任何倉位（直到 ADI 重新穩定）

**原因**：

> -40% 是系統性崩潰信號，不是正常波動
> 此時放大部位 = 自殺

---

### 情境 2：連續 -20% × 2 天

**觸發條件**：連續兩日累計跌幅 ≥ 20%

**系統反應**：

1. 計算累計損失：`N₁_new = N₁_old × 0.8 × 0.8`
2. 重新評估所有層級名義上限
3. 如果任何層級超過新規上限 → **強制減倉至新規上限**
4. ADI 重新計算，E 值可能大幅下降

---

### 情境 3：全市場高相關下跌 + 回升

**觸發條件**：帳面資金從高點回撤 ≥ 30%

**系統反應**：

1. **停止所有新倉建立**
2. 評估現有倉位是否仍在 ΔATR 上限內
3. 若超出 → 決定是否止損或持有
4. 等候 ADI 回升 + ATR 重新壓縮才能恢復操作

## 風險池配置（末日假設下）

### 總風險池

```
R_total = N₀ × 40% = 400 USDT
```

### L/C/T 風險分配

| 層級 | 比例 | 風險預算 | 可承受情節 |
|------|------|---------|-----------|
| **L** | 50% | 200 USDT | 結構性恐慌，可接受 |
| **C** | 30% | 120 USDT | 系統性回撤，需監控 |
| **T** | 20% | 80 USDT | 預設歸零，不干預 |

---

## 鐵律（不可違反）

### 1️⃣ 情緒禁止

> **在任何極端行情下，絕對不因為「感覺便宜」而加碼**

### 2️⃣ 系統比情緒重要

> **當你想違背系統規則時，系統永遠是對的**

### 3️⃣ 活着比賺錢重要

> **錯過 V 型反彈不會死，但一次失控加倉可能會**

### 4️⃣ E 值只下調，不上調

> **ADI 下降時 E 跟著下降，ADI 上升時 E 才可能上升**
> 防止在情緒高點放大風險

## 實戰練習題

### 情境

- N₀ = 1000
- N₁ = 850（已虧損 150）
- ADI = 20 → E = 0.68
- L 層持有名義 = 600

### 計算

1. L 層新規上限？
2. 是否需要減倉？
3. 可否新建 C 層倉位？

### 解答

```
1. L新規 = 1500 × (850/1000) = 1275
   → 持有 600 < 1275，無需減倉

2. 風險使用率 = 600/1275 = 47%
   → 🟢 安全，無需減倉

3. C層上限 = 900 × 0.85 = 765
   → 尚未使用，可建立，但受限於 E=0.68
   → 謹慎操作
```
