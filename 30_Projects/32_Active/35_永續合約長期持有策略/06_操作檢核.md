# 06_操作檢核清單

## 使用說明

> 每次建倉、補倉、加碼前，**必須逐項確認**。
> 只有全部 ✅ 才能執行，否則嚴禁操作。

---

## 建倉前檢核

### Step 1：身份確認

- [ ] 這個標的屬於 L/C/T 哪一層？
- [ ] 該層目前風險使用率？
- [ ] 是否符合該層的行為權限？

### Step 2：趨勢確認

- [ ] ADI 數值是多少？
- [ ] E（縮放係數）是多少？
- [ ] 是否在可操作範圍內（ADI > 20）？

### Step 3：空間確認

- [ ] 該標的名義上限？
- [ ] 建倉後風險使用率？
- [ ] 是否超過該層硬上限？

### Step 4：金額確認

- [ ] 單筆建倉金額 = N₁ × 10% × E
- [ ] 是否低於該標的名義上限？
- [ ] 計算結果：

```
目標標的：___________
層級：___________
ADI：___________
E：___________
單筆建倉額：___________ USDT
建倉後風險使用率：___________%
```

**全部 ✅ 才能下單**

---

## 補倉前檢核

### Step 1：層級確認

- [ ] 目標標的屬於 L/C/T 哪一層？
- [ ] L 層：結構是否未破？
- [ ] C 層：是否 ATR 擴張初期？
- [ ] T 層：是否**禁止補倉**？

### Step 2：ATR 確認

- [ ] 當前價格距離建倉價的 ΔATR 是多少？
- [ ] 是否滿足該層 ATR 上限？
- [ ] 是否 ≥ 1 ATR 回撤觸發條件？

### Step 3：行為確認

- [ ] 該層是否允許此次補倉？
- [ ] 補倉後該層風險使用率？
- [ ] 總帳戶在末日假設下是否可承受？

### Step 4：金額確認

- [ ] 單筆補倉金額 = N₁ × 5% × E
- [ ] 累積名義部位 = 建倉 + 歷次補倉
- [ ] 是否低於該標的名義上限？

### 計算模板

```
目標標的：___________
當前 ΔATR：___________（該層上限：___________）
單筆補倉額：___________ USDT
累積總名義：___________ USDT
該標的上限：___________ USDT
補倉後風險使用率：___________%
```

**全部 ✅ 才能下單**

## 風險警示燈號

| 燈號 | 風險使用率 | 操作建議 |
|------|-----------|----------|
| 🟢 綠 | 0–50% | 正常操作 |
| 🟡 黃 | 50–70% | 警戒，減少非必要加碼 |
| 🟠 橙 | 70–85% | 警告，停止所有新倉 |
| 🔴 紅 | 85–100% | 極限，禁止所有新倉 |
| ❌ 黑 | >100% | 違規，系統性風險 |

## 情緒檢核（最重要）

在執行任何操作前，回答這三個問題：

### 1️⃣ 我現在是因為「覺得便宜」而想操作嗎？

- 是 → ❌ 停止
- 否 → 繼續

### 2️⃣ 我現在是因為「不想錯過行情」而想操作嗎？

- 是 → ❌ 停止
- 否 → 繼續

### 3️⃣ 我現在是想「挽回之前的虧損」而想操作嗎？

- 是 → ❌ 停止
- 否 → 繼續

> **三個問題任一回答「是」，都必須強制停止操作。**

## 快速參考卡

```
【每次操作前】
1. 確認層級（L/C/T）
2. 確認 ADI → E 值
3. 確認 ΔATR 是否觸發
4. 計算金額（N₁ × % × E）
5. 確認不超過有名義上限
6. 情緒三問

【鐵律】
- N₀ 只在明確出入金時更新
- T 層永不補倉
- E 只下調，不上調
- 情緒「是」= 停止
```
