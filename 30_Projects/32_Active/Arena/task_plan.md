# Arena 追蹤系統建構計劃

**日期**：2026-04-14
**狀態**：規劃中
**定位**：依 Arena 框架 + Life OS 基底，建構日程/任務/專案的追蹤系統

---

## 一、系統定位

### 1.1 與 Arena 的關係

Arena 是一套「認識自己 → 規劃目標 → 策略行動 → 迭代執行」的完整框架。追蹤系統是 Arena 的**執行反饋層**，讓每次訪談和行動計劃都有數據支撐。

### 1.2 與 Life OS 的關係

Life OS 是 Arena 的**系統化工具層**，提供了：
- Goal → Project → Task 的三層串聯
- Area 五宮格的人生維度平衡
- 主頁 Components 的視覺化儀表板
- OS-每日檢查/OS-每週覆盤的日常運作機制

追蹤系統將整合 Arena 的「進化追蹤」與 Life OS 的「系統工具」，成為一個完整的執行反饋系統。

## 三、日程追蹤（🗓️ Schedule Layer）

### 3.1 每日追蹤機制

**組件**：OS-每日檢查.md + 每日節奏 Ritual

**追蹤內容**：
- 能量校準（🟢/🟡/🔴/⚫）
- 今日 MIT（最重要的一件事）
- HUMAN 3.0 象限覺察
- 任務完成度

**數據流向**：
```
每日檢查 → TaskNotes → Area 五宮格 → 主頁 Components
```

### 3.2 每日 Components（建議新增）

```dataview
TABLE WITHOUT ID file.link AS 日期, 能量, MIT, 完成度
FROM "20_Life_Planning/21_Journsal"
WHERE date(time_create) >= date(today) - 7
SORT date(time_create) DESC
```

## 四、任務追蹤（📋 Task Layer）

### 4.1 每週追蹤機制

**組件**：OS-每週覆盤.md + 每週覆盤 Ritual

**追蹤內容**：
- 30天行動計劃執行情形
- 任務完成率
- 障礙模式識別
- HUMAN 3.0 四象限進化比較

**數據流向**：
```
每週覆盤 → 進化追蹤摘要 → HUMAN_3.0評估報告 → Arena Stage 1
```

### 4.2 HUMAN_3.0 進化追蹤報告（已建構）

已更新的 `HUMAN_3.0_評估報告_模板.md` 包含：
- 進化追蹤摘要（維度比較）
- 行動計劃執行情形（前次清單 vs 本次結果）
- 障礙與干擾分析
- 下次訪談重點預先標記

### 4.3 任務 Components（建議新增）

```dataview
TABLE WITHOUT ID file.link AS 任務, status, projectId, goalId, 完成度
FROM "TaskNotes"
WHERE contains(tags, "life-os/task")
SORT status ASC, date(time_create) DESC
```

## 五、專案追蹤（📁 Project Layer）

## 八、風險與應對

| 風險 | 影響 | 應對 |
|------|------|------|
| Life OS 與 Arena 重疊 | 系統複雜度增加 | 以 Arena 為框架，Life OS 為工具層 |
| 追蹤數據過多 | 用戶負擔過重 | 精簡關鍵指標，只追蹤影響決策的數據 |
| Components 效能 | Dataview 查詢慢 | 限制查詢範圍、使用過濾 |
| 使用者堅持度 | 系統流於形式 | 設計簡單的 Ritual，降低進入門檻 |
