# Stage 2：規劃目標

> 這是 Arena 課程的第二個階段。你的任務是：基於 Stage 1 的自我認知，建立你想要的未來方向與具體路徑。
>
> **核心問題：我要去哪裡？**

---

## Stage 2 總覽

### 目標

基於 Stage 1 的自我認知地圖，探索你真正想要的方向，並將其轉化為可執行的路徑規劃。

### 與 Stage 1 的銜接

Stage 1 的每一個步驟，都需要你回頭參照 Stage 1 的產出。

### 流程

```
Step 1 → 識別問題層級（我的問題屬於哪個層次？）
Step 2 → 願景探索（我想要什麼？）
Step 3 → 前置條件審核（我的目標現實嗎？）
Step 4 → 差距分析（我和目標差多遠？）
Step 5 → 變現階梯定位（我的價值如何交付？）
Step 6 → 階段拆解（如何分步驟前進？）
Step 7 → 產出路徑圖
Step 8 → AI 訪談對齊（取得客觀視角）
```

### 產出

一份完整的「路徑規劃圖」，作為 Stage 3（策略規劃）的輸入。

## 2.0 前置說明

### 為什麼要先識別問題層級？

多數人在設定目標時，常犯的錯誤是：試圖用 Tier 3（成長）的解決方案，去處理一個 Tier 1（生存）的問題。

例如：一個人正處於財務危機，卻花大量時間在網上研究「如何找到人生使命」。他的大腦的神經系統會持續處於防禦模式，讓他無法真正思考長遠方向。

**Stage 2 的第一步，是確認你的問題在哪個層級。** 如果你在錯誤的層級努力，所有努力都會像沙中建樓。

---

## 2.1 問題層級識別

### 什麼是問題層級？

問題層級是一種優先順序框架。它說明：不同層級的問題有不同的「神經生物學優先順序」——解決低層級問題，是追求高層級目標的前提。

### 五層問題架構

| 層級 | 名稱 | 問題範例 | Arena 的回應 |
|------|------|----------|-------------|
| **Tier 1** | 生存問題 | 健康危機、財務緊急狀況、法律糾紛 | 必須優先解決，不計代價 |
| **Tier 2** | 穩定問題 | 基礎收入能力不足、情緒波動大、睡眠混亂 | 建立基本安全感與節奏 |
| **Tier 3** | 成長問題 | 職涯晉升遇到瓶頸、體能需要優化、方向需要釐清 | 開始主動發展 |
|| **整合問題** | 六商之間的平衡、系統建立、智慧修煉 | 追求同步發展 |
| **Tier 5** | 元問題 | 自主權的發展、非對抗性實踐、為更大的使命做出貢獻 | 參與更大的進化遊戲 |

### 為什麼層級順序不能跳過？

基於需求層次理論：只有當底層的安全感被滿足，大腦的前額葉皮質才能正常運作，進而開啟學習、創造、整合的高階功能。

簡單說：**在財務危機中談使命，就像在膝跳反射時做數學題。**

### 你的焦慮來自哪個層級？

自我檢查：過去三個月，你最常感到焦慮的是什麼？這個焦慮對應哪個層級？

| 層級 | 自我判斷 |
|------|---------|
| ☐ Tier 1（生存危機）| 如果你正面臨健康破產、財務破產或法律危機 |
| ☐ Tier 2（穩定問題）| 如果你的基本生活保障（收入、睡眠、情緒）還沒有穩定 |
| ☐ Tier 3（成長問題）| 如果你的基本生活穩定，但你在尋求突破 |
| ☐ Tier 4（整合問題）| 如果你的六商都在發展，但整合困難 |
| ☐ Tier 5（元問題）| 如果你已經有一定的發展基礎，現在想參與更大的使命 |

### 識別自己的主要問題層級

```
我當前最核心的問題層級：_______________

我的主要焦慮是：
1. _______________
2. _______________
3. _______________

對應層級：
1. _______________
2. _______________
3. _______________
```

### 重要提醒

> 如果你識別到自己在 Tier 1 或 Tier 2，先停下來。Stage 2 的剩餘步驟——願景探索、前置條件審核——都建立在一個前提上：你的基本生活有一定的穩定度。如果基本穩定都沒有，先回到 Stage 1 的商數修補，或者直接尋求專業協助（財務顧問、健康教練、心理諮商）。

## 2.2 願景探索

### 什麼是願景探索？

願景探索的核心工具是「奧德寨計畫」（Odyssey Plan）。這個工具強迫你的大腦離開「做還是不做」的二元思維，透過三個不同版本的五年規劃，讓你看到比單一選項多得多的可能性。

### 奧德寨計畫三版本

#### 計畫一：現有道路的延伸

**核心問題：** 如果你繼續現在的方向，最大可能性是什麼？

這個計畫不是要你「堅持現狀」，而是客觀分析：如果沒有任何外部改變，你的現有道路五年後會是什麼樣子？

**練習：**
- 標題：給這個版本的自己取一個五年後的標題（例如：「持續精進的資深工程師，開始帶團隊」）
- 五年願景：描述這個版本的你在哪裡、在做什麼
- 儀表板評估：
  - 資源（時間、金錢、技能、人脈）：___
  - 喜愛度：你有多喜歡這個方向？___
  - 信心度：你有多確信這個方向可行？___
  - 一致性：這個方向與你的價值觀有多契合？___

#### 計畫二：如果現在的消失了

**核心問題：** 如果這個方向突然消失——產業被顛覆、你所依賴的技能突然過時、你目前的位置不存在了——你會做什麼？

這個計畫是「備用計畫」的心理建設，也是一種壓力測試：如果你不能繼續現在的方式，你真正的核心能力是什麼？

**練習：**
- 標題：_______________
- 五年願景：_______________
- 儀表板評估：
  - 資源：___ 喜愛度：___ 信心度：___ 一致性：___

#### 計畫三：如果錢不是問題

**核心問題：** 如果金錢和他人眼光都不重要，你會做什麼？

這個版本是「狂想版」——允許自己完全離開現實約束，大膽探索內心最深處的渴望。

**練習：**
- 標題：_______________
- 五年願景：_______________
- 儀表板評估：
  - 資源：___ 喜愛度：___ 信心度：___ 一致性：___

### 價值主張與傷痕分析

除了奧德寨計畫，你還需要思考：你要用什麼方式創造並交付價值？

**傷痕分析法**

> 「你的痛苦在哪裡，你的解決能力就在哪裡。」

這個方法說的是：你曾經克服過的困難，讓你具備了獨特的解決問題的能力。你要做的，是把這個能力轉化為可以交付的價值。

```

1. _______________
2. _______________

對應的解決能力：
1. _______________
2. _______________

我可以幫助誰解決這個問題：
1. _______________
2. _______________
```

三個版本的奧德寨計畫都完成之後，回答這個問題：

**「如果我只能專注一個方向，選擇哪一個？為什麼？」**

## 2.3 前置條件審核

### 什麼是前置條件審核？

Arena 的前置條件審核借鑒了遊戲邏輯：在遊戲中，你想解鎖某個成就，需要先達到一定的等級和技能點數。現實世界也是一樣——如果你想達成某個目標，你需要先確認自己的商數等級是否足夠支撐這個目標。

**現實的目標設定：目標 → 執行**
**Arena 的目標設定：目標 → 審核前置條件 → 差距分析 → 順序建議 → 執行**

如果你的商數等級不足以支撐你的目標，硬衝只會讓你burnout。

### 商數等級需求對照

不同的目標類型，對各商數有不同的等級需求：

| 目標類型 | HQ | EQ | XQ | PQ | AQ | FQ |
|----------|-----|-----|-----|-----|-----|-----|
| 穩定的日常工作 | 1.5 | 1.5 | 1.5 | 1.0 | 1.0 | 1.5 |
| 建立副業/興趣變現 | 1.8 | 2.0 | 1.8 | 1.5 | 1.5 | 2.0 |
| 創業/自由職業 | 2.0 | 2.2 | 2.0 | 2.0 | 2.0 | 2.5 |
| 系統化規模化 | 2.0 | 2.5 | 2.5 | 2.5 | 2.5 | 3.0 |

## 2.4 差距分析

### 什麼是差距分析？

差距分析是計算「你想要的」和「你現在的」之間的距離。這不是讓你焦慮的數據，而是規劃路徑的起點。

### 六商差距計算

```
目標：抵達 [商數] 等級 [Y]
當前： [商數] 等級 [X]
差距： [Y - X]
```

| 商數 | 當前等級 | 目標需求 | 差距 | 優先級 |
|------|----------|----------|------|--------|
| HQ |  |  |  | |
| EQ |  |  |  | |
| XQ |  |  |  | |
| PQ |  |  |  | |
| AQ |  |  |  | |
| FQ |  |  |  | |

### 優先順序邏輯

差距分析告訴你「差多遠」，但優先順序告訴你「先走哪條路」。

**優先順序的三個原則：**

1. **先發展差距最小的商數** — 建立信心，避免一開始就被挫折打擊
2. **再發展對目標影響最大的商數** — 加速前進，集中火力突破瓶頸
3. **同時發展支持性商數** — 建立系統支持，避免單點突破後全盤崩潰

**典型優先順序範例：**

```
目標：創業（FQ 需要 2.5 等級）

前置條件審核：
  - HQ 需要 2.0（當前 1.3，缺 0.7）→ 落後明顯
  - EQ 需要 2.2（當前 2.1，缺 0.1）→ 差距最小，先建立信心
  - XQ 需要 2.0（當前 1.5，缺 0.5）→ 需要支援系統
  - PQ 需要 2.0（當前 1.0，缺 1.0）→ 需要支援系統
  - AQ 需要 2.0（當前 1.2，缺 0.8）→ 核心目標
  - FQ 需要 2.5（當前 1.8，缺 0.7）→ 核心目標

執行順序：
  1. 先發展 EQ（差距最小，先建立信心）
  2. 再發展 FQ（核心目標）
  3. 同時發展 XQ/PQ（支持系統）
  4. 最後發展 HQ（需要前者的能量支持）
```

## 2.5 變現階梯

### 什麼是變現階梯？

變現階梯描述的是一人企業從服務到產品的演進路徑。如果你的 Stage 1 原型識別或 Stage 2.2 的傷痕分析指向了一個可以創造價值的方向，變現階梯幫助你找到適合的貨幣化策略。

### 三階段變現路徑

#### Stage 1：實證服務（PoC）

**核心：** 用技能換取現金，收集痛點，驗證你的方法論是否有效

| 項目 | 內容 |
|------|------|
| 變現方式 | 顧問、教練、自由接案 |
| 時間槓桿 | 低（時間換錢） |
| 驗證目標 | 確認目標客戶的痛點真實存在，你的方法論有效 |
| 收入天花板 | 有限（受限於你的個人時間） |

#### Stage 2：經驗模式化

**核心：** 將你的解決方案流程化、模板化，變成可以規模化交付的產品

| 項目 | 內容 |
|------|------|
| 變現方式 | 課程、訓練營、模板、數位產品 |
| 時間槓桿 | 中（一次製作，多次銷售） |
| 驗證目標 | 確認你的產品可以規模化交付 |
| 收入天花板 | 擴大（但仍需主動交付） |

#### Stage 3：百萬美元產品

**核心：** 建立系統，讓收入與你的時間完全脫鉤

| 項目 | 內容 |
|------|------|
| 變現方式 | 訂閱制、會員制、被動收入、聯盟行銷 |
| 時間槓桿 | 高（系統為你工作） |
| 驗證目標 | 建立持續、被動的收入流 |
| 收入天花板 | 擴大（被動收入） |

### 你在變現階梯的哪個位置？

```
你目前的變現狀態：
☐ Stage 1（實證服務）— 剛開始用技能變現
☐ Stage 2（經驗模式化）— 已有可規模化的產品
☐ Stage 3（百萬美元產品）— 已有被動收入系統

你的下一個目標台階：
☐ 從 Stage 1 前進到 Stage 2
☐ 從 Stage 2 前進到 Stage 3
☐ 其他：_______________
```

### 變現方式的選擇

| 方式 | 適合階段 | 時間槓桿 |
|------|----------|----------|
| 顧問/教練 | Stage 1 | 低 |
| 課程/訓練 | Stage 2 | 中 |
| 會員制 | Stage 3 | 高 |
| 數位產品 | Stage 2-3 | 高 |
| 聯盟行銷 | Stage 3 | 高 |
| 授權/IP | Stage 3 | 高 |

## 2.6 階段拆解

### 什麼是階段拆解？

階段拆解是將你的大目標轉化為「可執行的台階」——每個台階是一個相對較小的子目標，完成後可以進入下一個台階，最終到達大目標。

### 進化台階設計

Arena 的階段拆解採用 30-90-180 天的時間框架，每個台階的長度代表不同的發展階段：

**30 天（第一階段）— 識別與啟動**

```
目標：識別並開始發展核心商數
子目標：[商數] 等級 從 X.X → X.X
關鍵行動：
  - _______________
  - _______________
里程碑：[可衡量的進步]
```

**90 天（第二階段）— 鞏固與擴展**

```
目標：鞏固並擴展
子目標：[商數] 等級 從 X.X → X.X，[商數] 等級 從 Y.Y → Y.Y
關鍵行動：
  - _______________
  - _______________
里程碑：[可衡量的進步]
```

**180 天（第三階段）— 整合與驗證**

```
目標：整合並驗證
子目標：[商數] 等級 從 X.X → 目標值
所有商數達到前置條件
里程碑：[系統化/原型驗證]
```

### 等級進化檢查點

在每個台階結束時，用這個檢查問題評估自己是否真的升級了：

| 等級 | 檢查問題 |
|------|----------|
| **1.5** | 我開始主動管理這個商數，不再完全依賴外部指令？ |
| **2.0** | 我建立了屬於自己的系統，開始追求差異化？ |
| **2.5** | 我開始整合多元視角，有了系統性的思維？ |
| **3.0** | 我能同時持有多個矛盾觀點，策略性地選擇應對方式？ |

## 2.8 AI 訪談對齊

### 為什麼需要 AI 訪談？

目標設定是人的主觀判斷，受情緒、願望焦慮和自我防衛機制影響。AI 訪談提供邏輯一致的外部視角，檢驗你的目標設定背後的假設是否合理。

### AI 訪談在做什麼？

Arena 的 AI 會：

1. **驗證目標與自我認知的一致性**（你說你想創業，但你的 FQ 是 1.8，距離創業需求的 2.5 還差很遠——你有這個心理準備嗎？）
2. **檢查優先順序的邏輯**（你說要先發展 FQ，但 FQ 是差距最小的商數——為什麼不是直接解決落後最大的商數？）
3. **挑戰模糊的目標描述**（「我想財務自由」不是目標，「我想在兩年內建立一個每月被動收入 3 萬元的訂閱制產品」才是目標）
4. **識別可能的美麗假話**（你的願景描述得很好，但你的行動計劃是否真的能到達那裡？）

### AI 訪談觸發條件（教練/Agent 專用）

當學員的回答觸發以下條件時，Agent 應主動追問：

**① 當學員的目標商數需求與當前等級差距過大時（> 1.0 等級）：**
> 「你設的目標需要 [商數] 等級 [X]，但你目前是 [Y]。這個差距需要在 [時間] 內補足。你有什麼具體計劃可以加速這個過程？」

**② 當學員的目標描述模糊，無法量化時：**
> 「請用具體數字描述你的目標——多少時間、多少收入、影響多少人？」

**③ 當學員設定的時間框架與台階不匹配時：**
> 「你想在 30 天內達成 [目標]，但你的現況距離這個目標還差 [X.X] 等級。這個時間設定現實嗎？」

**④ 當學員的優先順序與差距分析結果矛盾時：**
> 「你的差距分析顯示 [商數] 落後最多，但你的優先順序是先發展 [商數]。可以解釋一下這個選擇嗎？」

**⑤ 當學員的變現路徑與其意識層級不匹配時：**
> 「你想直接跳到 Stage 3（被動收入），但你目前還沒有通過 Stage 1 驗證的痛點和方法論。為什麼不先建立這個基礎？」
