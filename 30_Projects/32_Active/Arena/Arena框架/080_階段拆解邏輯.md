# 080_階段拆解邏輯

> Arena 的大目標→階段目標拆解系統 — 融合設計人生 × 遊戲化機制
> 產出時間：2026-04-01

note: Arena 課程產品的框架組成文件

## 融合說明

Arena 的階段拆解結合了：
- 設計人生的奧德賽計畫（三版本概念）
- 遊戲化的維度等級前置條件審核

---
note: Arena 課程產品的框架組成文件

## 核心流程

```
1. 識別大目標
   ↓
2. 前置條件審核（維度等級需求）
   ↓
3. 差距分析
   ↓
4. 選擇發展路徑（奧德賽三版本）
   ↓
5. 設定階段目標（30天 / 90天 / 180天）
   ↓
6. 建立里程碑
```

---
note: Arena 課程產品的框架組成文件

## 奧德賽計畫的三版本

| 版本 | 問題 | 用途 |
|------|------|------|
| 計畫一 | 你現有的道路 | 現狀延伸 |
| 計畫二 | 如果現在的消失 | 備用/軸轉 |
| 計畫三 | 如果錢不是問題 | 狂想目標 |

三個版本能讓我們的大腦自由地在不同可能性之間跳轉，發現單一方案中看不到的機會。

## Arena 的階段模板

### 30 天（第一階段）

**目標**：識別並開始發展核心維度

```
[維度] 維度：當前 [X] → 目標 [X+0.3]
關鍵行動：
  - [行動 1]
  - [行動 2]
里程碑：[可衡量的進步]
```

### 90 天（第二階段）

**目標**：鞏固並擴展

```
[維度] 維度：目標 [X+0.3] → [X+0.6]
[維度] 維度：目標 [Y] → [Y+0.2]
關鍵行動：
  - [行動 1]
  - [行動 2]
里程碑：[可衡量的進步]
```

### 180 天（第三階段）

**目標**：整合並驗證

```
[維度] 維度：目標 [X+0.6] → [目標值]
所有維度達到前置條件
里程碑：[系統化/原型驗證]
```

## 階段拆解範例

```
大目標：創業（價值 2.5）
  ↓
前置條件：身體 2.0 / 心智 2.2 / 關係 2.0
  ↓
差距：身體缺 0.7 / 心智缺 0.1 / 關係缺 1.0
  ↓
選擇路徑：計畫一（現有道路延伸）
  ↓
30天目標：心智 +0.2，建立每日反思習慣
90天目標：價值 +0.3，開始小規模變現
180天目標：身體 +0.3 / 關係 +0.3，達到創業前置條件
```

## 與 Stage 2 的對應

Stage 2 的核心任務：
1. 大目標建立
2. 前置條件審核（見 [[070_前置條件審核]]）
3. 差距分析
4. 階段拆解
5. 執行順序建議

## 使用時機

階段拆解在 Stage 2 完成前置條件審核後使用：
1. 根據差距分析結果
2. 選擇適合的發展路徑
3. 設定 30-90-180 天的階段目標
4. 為每個階段設定里程碑
