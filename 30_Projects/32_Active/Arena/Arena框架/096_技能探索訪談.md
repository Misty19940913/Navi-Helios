# 096_技能探索訪談

> Arena Stage 2 的技能探索專屬訪談模組

## 融合說明

本檔案從 Skill Graph 概念萃取技能探索的自適應訪談機制，包括：
- 經驗回顧式問題設計（幫助不知道自己技能的人挖掘）
- 技能分類框架（Technical / Domain / Execution）
- 可轉移技能識別問題
- 技能缺口自檢問題
- 技能槓桿評估

## Phase 0: 技能探索引導

### 標準引導語

```
在我們進入目標規劃之前，我想先了解你目前的能力現況。

很多人對自己的技能沒有清晰的認知——他們以為自己「只是做了工作」，
卻沒有發現自己其實已經累積了有價值的技能。

我會引導你從過去的經驗中挖掘你的技能地圖。

請回想過去一年，你做過的專案、任務、或任何你認為有意義的工作。
不需要是正式的專案，任何讓你花費時間和精力的事都可以。

開始吧。
```

## Phase 1: 經驗回顧 → 技能萃取

### 核心問題（必問）

#### Q1: 經驗列舉

```
請列出過去一年你做過的所有專案、任務、或重要工作。
不需要完整，只要你能想起的都可以。

格式：
1. [專案/任務名稱] — [你做的主要事情] — [花了多少時間]
```

#### Q2: 深度挖掘（每個列舉的專案追問）

| 追問 | 目的 |
|------|------|
| 「你在這個專案中解決了什麼問題？」 | 問題識別能力 |
| 「你用什麼工具或方法完成的？」 | 技能線索 |
| 「如果讓你再做一次，你會哪裡不一樣？」 | 反思能力 |
| 「這個專案帶給你什麼改變？」 | 成長識別 |

#### Q3: 隱性技能識別

| # | 問題 | 目的 |
|---|------|------|
| Q3-1 | 「過去一年，有人請你幫忙什麼事嗎？他們為什麼找你？」 | 被需要的技能 |
| Q3-2 | 「什麼事情你做起來比大多數人輕鬆？」 | 天然優勢 |
| Q3-3 | 「什麼事情你需要花很多力氣，但多數人覺得簡單？」 | 需強化技能 |
| Q3-4 | 「當你感到『進入狀態』時，通常在做什麼？」 | Flow 觸發活動 |

## Phase 2: 技能分類

### 三類技能定義

| 類型 | 定義 | 特徵 |
|------|------|------|
| **Technical（技術）** | 可工具化、可測試、有標準流程的技能 | 寫代碼、設計、機械操作、數據分析 |
| **Domain（領域）** | 特定行業/主題的專業知識 | 財務知識、法律知識、行業經驗 |
| **Execution（執行）** | 與人協作、推動結果的能力 | 專案管理、教練、銷售、領導力 |

### 分類問題

```
根據你列舉的經驗和識別的技能，請將它們分類：

Technical 技能：
-

Domain 技能：
-

Execution 技能：
-

哪個類別讓你最有成就感？
哪個類別讓你覺得最困難？
```

### 等級追問

| 類型 | 追問 |
|------|------|
| Technical | 「你這個技能可以教給別人嗎？還是你自己也在學習中？」 |
| Domain | 「你的領域知識主要來自書本、經驗，還是兩者都有？」 |
| Execution | 「你通常是自己完成工作，還是透過他人完成？」 |

## Phase 3: 技能層級評估

### 技能熟練度量表

| 等級 | 定義 | 行為表現 |
|------|------|---------|
| **Lv1 入門** | 知道概念，未親自實踐 | 「我了解這是什麼，但沒做過」 |
| **Lv2 應用** | 能在指導下完成 | 「有人帶的時候我可以做」 |
| **Lv3 獨立** | 能獨立完成，常規任務 | 「我自己可以處理，常見問題都能解決」 |
| **Lv4 專精** | 能處理複雜、非標準情況 | 「困難的我也能處理，複雜情況有辦法」 |
| **Lv5 傳授** | 能教導他人，形成方法論 | 「我能總結出方法，讓別人也能做到」 |

### 評估問題

```
針對你識別出的每個技能，請回答：

1. [技能名稱] — 等級（1-5）— 證據（什麼讓你這樣認為？）
   等級：
   證據：

2. [技能名稱] — 等級（1-5）— 證據
   等級：
   證據：
```

### 技能雷達圖產出

```
根據評估結果，繪製技能雷達圖：

* 以主要技能為定錨點

## Phase 4: 可轉移技能識別

### 轉移技能定義

可轉移技能 = 能在多個領域使用的通用能力。

### 常見可轉移技能清單

| 技能類型 | 說明 |
|----------|------|
| 問題解決 | 分析問題並設計解決方案 |
| 溝通表達 | 清晰傳達想法，說服他人 |
| 專案管理 | 規劃、執行、按時交付 |
| 學習能力 | 快速掌握新領域知識 |
| 適應能力 | 在變化中保持效率 |
| 團隊協作 | 與他人有效合作 |
|  時間管理 | 優先排序，高效執行 |

### 識別問題

| # | 問題 | 目的 |
|---|------|------|
| T1 | 「你做過哪些不同領域的專案？他們有什麼共同點？」 | 共同技能識別 |
| T2 | 「你在 A 領域用到的方法，能用在 B 領域嗎？」 | 轉移測試 |
| T3 | 「什麼技能是你在每份工作中都會用到的？」 | 核心轉移技能 |
| T4 | 「你曾經把一個領域的技能用到完全不同的領域嗎？」 | 轉移成功案例 |

### 轉移路徑圖

```
可轉移技能：問題解決
    │
    ├── 從：金融分析
    ├── 到：營運優化
    └── 到：產品策略

可轉移技能：溝通表達
    │
    ├── 從：業務簡報
    ├── 到：內部培訓
    └── 到：內容創作
```

## Phase 5: 技能缺口自檢

### 缺口分類（參考 Skill Graph）

| 缺口類型 | 定義 | 緊急性 |
|-----------|------|--------|
| **Technical 缺口** | 目標需要的技術能力你沒有 | 高（影響執行）|
| **Domain 缺口** | 目標需要的領域知識你不熟悉 | 中（需要學習）|
| **Execution 缺口** | 目標需要的管理/協作能力你不足 | 中（影響規模）|

### 缺口分析問題

| # | 問題 | 目的 |
|---|------|------|
| G1 | 「你的目標需要什麼技能？」 | 目標技能清單 |
| G2 | 「這些技能你現在有多少？」 | 現有技能對照 |
| G3 | 「缺口最大的是哪個？」 | 優先填補 |
| G4 | 「填補這個缺口需要多少時間？」 | 時間規劃 |
| G5 | 「你自己可以學，還是需要外部資源？」 | 資源規劃 |

### 技能缺口矩陣

```
產出格式：

目標：[描述你的目標]

所需技能 vs 現有技能：

| 技能 | 目標需要等級 | 現有等級 | 缺口 | 填補方式 |
|------|-------------|---------|------|---------|
| Python | Lv3 | Lv1 | -2 | 線上課程 + 專案練習 |
| 數據分析 | Lv3 | Lv2 | -1 | 在職練習 |
| 專案管理 | Lv3 | Lv3 | 0 | 維持 |
```

## Phase 6: 技能槓桿評估

### 槓桿技能定義

高槓桿技能 = 用最少時間投入，獲得最大價值回報的技能。

### 評估商數

| 商數 | 問題 |
|------|------|
| **市場價值** | 這個技能在市場上值多少？（薪資/收費） |
| **可擴展性** | 一單位時間能服務多少人？（1人 vs 1000人） |
| **組合效應** | 這個技能能和哪些技能組合放大價值？ |
| **入門門檻** | 掌握這個技能需要多少時間？ |
| **持久性** | 這個技能5年後還有價值嗎？ |

### 高槓桿技能篩選問題

| # | 問題 |
|---|------|
| L1 | 「什麼技能你已經具備，且市場需求高？」 |
| L2 | 「什麼技能你一小時的工作可以幫助100人？」 |
| L3 | 「什麼技能可以和其他技能組合，變成更高價值的服務？」 |
| L4 | 「如果只能選一個技能精進，你會選哪個？為什麼？」 |

### 高槓桿技能優先矩陣

```
產出格式：

---------|------|----------------|----
1        |      |                |

## 自適應分支邏輯

### 根據用戶回應自動調整問題

#### 情況A：用戶說「我不知道自己有什麼技能」

**觸發分支：**
- Q1 替換為：「回想過去一個月，你被要求做過哪些事？」
- 追加 Q3-1：「別人通常因為什麼找你幫忙？」
- 追加 Q3-2：「什麼事情你做起來比多數人輕鬆？」

#### 情況B：用戶有豐富經驗但無法提煉

**觸發分支：**
- 提問：「你最近一個月最花時間的事是什麼？」
- 拆解：「這個事情裡，你做了哪些具體動作？」

#### 情況C：用戶有明確目標

**觸發分支：**
- 跳過 Phase 1/2，直接進入 Phase 5（技能缺口自檢）
- 追加：「你認為這個目標需要什麼技能？」
- 「你現在距離這個目標還差什麼？」

## AI 訪談約束

| 約束 | 說明 |
|------|------|
| 從經驗出發 | 不直接問「你會什麼」，而是從「你做過什麼」引導 |
| 深度追問 | 每個答案都要追問「能舉個例子嗎？」 |
| 具體化 | 抽象技能需要具體行為支撐 |
| 不假設 | 不假設用戶知道自己擅長什麼 |
| 優先順序 | 技能很多時，幫用戶聚焦最重要的 |
