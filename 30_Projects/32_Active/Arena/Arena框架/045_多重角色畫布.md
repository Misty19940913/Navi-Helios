# 045_多重角色畫布

> 六商視角下的多重角色分析工具
> 產出時間：2026-04-27
> 整合：BMY 多重角色畫布 × Arena 六商系統

---

## 融合說明

本畫布將 BMY 多重角色畫布整合至 Arena 六商系統，用於 Stage 1 現況評估與 Stage 2 目標規劃。

核心問題：我正在扮演哪些角色？每個角色如何調動六商資源？哪些角色被忽略？

---

## 用途

1. **暴露角色失衡** — 某些角色消耗單一商數過度，其他商數閒置
2. **識別角色衝突** — 不同角色對同一商數有相反需求
3. **優化角色組合** — 刪除空洞角色，強化核心角色
4. **對齊 Arena Stage** — 角色掙扎往往是 Stage 推進的切入點

---

## 第一部分：角色清單

### 我的核心角色

列出你目前投入時間的所有角色（工作/家庭/興趣/責任）。

```
角色名稱        | 每天投入時間 | 激情程度（1-5）| 疲憊程度（1-5）
--------------------------------------------------------------
例：公務人員    |              |                  |
例：父親/母親   |              |                  |
例：內容創作者  |              |                  |
例：健身者      |              |                  |
例：閱讀者      |              |                  |
```

### 角色分類

```
┌─────────────────────────────────────────────────────┐
│  謀生角色（時間最多，但不一定最有意義）                │
│  例：員工、的自由工作者、創業者                        │
├─────────────────────────────────────────────────────┤
│  關係角色（情感支持與被需要的來源）                    │
│  例：父母、伴侶、子女、朋友                           │
├─────────────────────────────────────────────────────┤
│  成長角色（技能與認知發展）                           │
│  例：學生、閱讀者、課程學習者                         │
├─────────────────────────────────────────────────────┤
│  娛樂角色（恢復與快樂）                               │
│  例：遊戲玩家、業餘音樂人、釣魚                       │
└─────────────────────────────────────────────────────┘
```

## 第二部分：角色 × 六商 消耗地圖

每個角色消耗哪些商數最多？圈出主要消耗，標註程度。

```
         │ FQ  │ HQ  │ EQ  │ PQ  │ AQ  │ XQ  │
角色名稱 │ 財富│ 健康│ 情緒│ 玩商│ 逆境│ 覺察│
---------|-----|-----|-----|-----|-----|-----|
公務人員 │ ●●  │ ●   │ ●●● │     │ ●   │     │
父親     │     │     │ ●●● │ ●   │     │ ●●  │
內容創作者│ ●● │     │ ●●  │ ●●  │ ●●  │ ●   │
健身者   │     │ ●●● │     │ ●   │     │     │
```

**符號說明**：● 主要消耗 / ◐ 次要消耗 / 空白 幾乎不消耗

## 第三部分：角色衝突診斷

### 商數過度集中

檢查是否有單一商數被過多角色共同消耗：

```
問題訊號：
□ 某商數被 3+ 角色同時標記為主要消耗
□ 感覺「每天都在消耗同一個商數」
□ 某商數長期處於低等級（感覺「耗竭」）

集中範例：
  FQ 被 公務人員 + 內容創作者 + 投資者 同時大量消耗
  → 收入多元化可能緩解，但根本原因是角色組合失衡
```

### 角色互相矛盾

檢查不同角色對同一商數的期待是否矛盾：

```
衝突範例：
  「創業者」需要 FQ 3.0 等級（冒險投資）
  「穩定者」需要 FQ 1.5 等級（保守理財）
  兩者同時存在 → 需要明确哪個是當前主角色

診斷問題：
□ 我在兩個角色之間拉扯嗎？
□ 每個角色對同一商數的期待是否一致？
□ 哪些角色可以合併？哪些必須放棄？
```

## 第四部分：角色進化地圖

### 角色階段演變

每個角色都會經過類似 Stage 的進化歷程：

```
Stage 1（覺察）：我為什麼要扮演這個角色？它是否服務於我的目標？
Stage 2（選擇）：我选择這個角色的程度有多高？是被迫還是主動？
Stage 3（深化）：如何讓這個角色成為六商發展的資產而非負擔？
```

### 角色與六商進化

```
角色升級時機判斷：

1. 當角色開始輕鬆愉快地調動多個商數 → 該角色進入 Stage 2
2. 當角色成為其他角色的助力而非阻力 → 該角色進入 Stage 3
3. 當角色不再需要意志力維持 → 該角色已整合進生活系統

角色降級警訊：

1. 每次想到這個角色就感到疲憊 → Stage 1 未完成
2. 必須用額外休息來補償這個角色 → 能量邊界被破壞
3. 這個角色破壞了其他角色的功能 → 角色衝突未解決
```

## 第五部分：理想角色組合

### 根據六商平衡需求選擇角色

```
如果你的 HQ（健康商）較低：
→ 選擇強調身體使用的角色（健身教練、舞蹈、園藝）
→ 避免只需要坐著的角色

如果你的 EQ（情緒商）較低：
→ 選擇需要深度人際互動的角色（教練、導師、團隊負責人）
→ 避免只需要單打獨鬥的角色

如果你的 PQ（玩商）較低：
→ 選擇具有內在樂趣的角色（音樂、遊戲、業餘嗜好）
→ 避免只有外在目標的角色

如果你的 AQ（逆境商）較低：
→ 選擇有適度挑戰的角色（比賽、競選、公開演講）
→ 避免過度穩定或過度刺激的角色
```

### 六商平衡的理想角色組合

```
最佳實踐：六商都有對應角色覆蓋

範例角色清單（覆蓋全部六商）：
- FQ：創業者 / 投資者 / 自由工作者
- HQ：健身者 / 運動員 / 舞蹈者
- EQ：父母 / 伴侶 / 團隊領袖 / 教練
- PQ：遊戲玩家 / 音樂人 / 業餘創作者
- AQ：創業者 / 競賽者 / 改革者
- XQ：作者 / 導師 / 哲學實踐者

診斷：我的角色清單是否覆蓋了所有六商？
     哪些商數完全沒有對應角色？
```

---

## 第六部分：角色斷捨離決策

### 決策框架

```
必須保留的角色：
□ 這個角色服務於我的核心目標
□ 這個角色滋養而非消耗我的主要商數
□ 這個角色帶來意義感而非只是責任

可以改造的角色：
□ 這個角色有價值但當前模式有問題
□ 調整期待、邊界或投入時間
□ 從「被迫角色」轉為「選擇角色」

果斷放下的角色：
□ 這個角色已經不服務於當前目標
□ 繼續扮演會消耗關鍵商數
□ 放下不等於失敗——是策略性配置
```

## 與 040_生活方式原型 的連結

多重角色畫布是 Lifestyle Archetype 的診斷工具。

```
工作狂原型
→ 角色症狀：幾乎只有「謀生角色」，其他角色空洞
→ 干預切入點：增加 PQ（玩商）和 EQ（關係）角色

探索者原型
→ 角色症狀：過多「成長角色」，缺乏「謀生角色」
→ 干預切入點：選擇一個成長角色轉化為謀生角色

整合者原型
→ 角色症狀：各類角色均有且相互支持
→ 干預切入點：維持平衡，預防角色崩潰
```

---

## 與 Arena Stage 的對應

```
Stage 1（覺察現況）
→ 完成「角色清單」和「角色×六商消耗地圖」
→ 識別角色失衡與衝突

Stage 2（目標對齊）
→ 根據六商進化需求，選擇要強化的角色
→ 決定要果斷放下的角色
→ 設計角色進化路徑

Stage 3（策略建構）
→ 將角色需求轉化為每週行動
→ 建立角色切換的能量邊界

Stage 4（迭代執行）
→ 定期審視角色組合與六商平衡
→ 根據進化階段調整角色比重
```
