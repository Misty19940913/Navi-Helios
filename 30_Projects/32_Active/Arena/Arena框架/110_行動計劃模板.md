# 110_行動計劃模板

> Arena Stage 3 的行動計劃框架 — 將策略轉化為可執行行動
> 融合來源：設計人生 × 商業模式 × Arena 框架
> 產出時間：2026-04-01

---

## 融合說明

| 來源 | 提供 |
|------|------|
| 設計人生 | PDCA 循環、Checkpoint、失敗免疫力 |
| 商業模式 | 關鍵活動、里程碑、關鍵指標 |
| Arena 100 | 路徑設計的台階規劃 |
| Arena 105 | 資源盤點的夥伴與資源 |

## 行動計劃總流程

```
Step 1: 定義行動類型
Step 2: 設計 PDCA 循環
Step 3: 設定 Checkpoint 評估點
Step 4: 建立失敗分類系統
Step 5: 產出 30 天行動計劃
```

## Step 3: 設定 Checkpoint 評估點

### Checkpoint 設計

| Checkpoint | 時間 | 評估商數 | 決策 |
|------------|------|----------|------|
| 週檢查 | 每週 | 當週進度 | 調整下週行動 |
| 月檢查 | 每月 | 月度目標 | 調整月度計劃 |
| 季評估 | 每季 | 階段目標 | 調整進化台階 |
| 年回顧 | 每年 | 年度願景 | 重新規劃 |

### Checkpoint 評估問題

#### 週檢查問題

| 商數 | 問題 |
|------|------|
| 進度 | 「這週完成了什麼？與目標相比如何？」 |
| 能量 | 「這週的能量狀態如何？什麼帶來能量？什麼消耗能量？」 |
| 學習 | 「這週學到了什麼？」 |
| 調整 | 「下週有什麼需要調整？」 |

#### 月檢查問題

| 商數 | 等級對照 |
|------|----------|
| 商數進步 | 「這個月每個商數有進步嗎？」 |
| 台階進度 | 「離下一個台階還有多遠？」 |
| 假設驗證 | 「原型測試的結果如何？」 |
| 計劃調整 | 「月度計劃需要調整嗎？」 |

#### 季評估問題

| 問題 | 目的 |
|------|------|
| 「這季的目標達成情況？」 | 成效檢視 |
| 「商數等級有變化嗎？」 | 等級追蹤 |
| 「策略是否有效？」 | 方向檢查 |
| 「下一季的重點？」 | 策略調整 |

## Step 4: 建立失敗分類系統

### 失敗類型

| 類型 | 說明 | Arena 處理 |
|------|------|-------------|
| **簡單錯誤** | 可立即修正的疏忽 | 立即修正 |
| **弱點觸發** | 個人限制被引發 | 覺察並管理 |
| **成長機會** | 有明確解決方案的挑戰 | 學習克服 |
| **系統失敗** | 外部因素導致 | 調整系統 |

### 失敗分類流程

```
1. 識別失敗事件
   「發生了什麼？」
   
2. 分類失敗類型
   「這是哪種類型的失敗？」
   
3. 提取教訓
   「我學到了什麼？」
   
4. 應用學習
   「這個教訓如何影響下次的行動？」
```

### 失敗記錄模板

```
【失敗事件】
日期：
事件描述：

【失敗分類】
類型：

【根本原因分析】
-

【學到的教訓】
-

【應用於未來】
下次遇到類似情況，我會：
```

### 失敗免疫力建立

| 步驟 | 行動 |
|------|------|
| 1. 記錄失敗 | 每次失敗後立即記錄 |
| 2. 重構框架 | 將失敗視為學習數據 |
| 3. 分享受控 | 與信任的人分享失敗 |
| 4. 建立支持 | 知道有人支持你面對失敗 |
