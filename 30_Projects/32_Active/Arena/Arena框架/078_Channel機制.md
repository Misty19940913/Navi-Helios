# 078_Channel 機制

> Arena 的 Channel 系統 — 融合 HUMAN 3.0
> 產出時間：2026-04-02

---

## 融合說明

本檔案從 HUMAN 3.0 (003) Part III Channel Mechanics 提取，用於：
- Stage 1 識別用戶是否正在經歷 Channel
- Stage 2-3 提供 Channel 進入/退出策略
- AI 報告中的特殊情況處理

---

## 核心定義

Channel 是高強度的發展加速期，在特定象限內的強迫性專注，能將數年的成長壓縮到數月。

---

## Channel 觸發信號

### 進入信號

| 信號 | 描述 |
|------|------|
| **時間扭曲** |  Flow 狀態（數小時感覺像幾分鐘）|
| **強迫性專注** | 無法停止學習/建構某主題 |
| **身體能量激增** | 生理能量湧現 |
| **想法 flood** | 靈感/下載不斷 |
| **社交退縮需求** | 需要獨處來專注 |
| **睡眠模式改變** | 失眠或高度專注後的崩潰 |
| **食慾波動** | 忘記吃飯或強迫性進食 |

### 自然觸發因素

| 觸發 | 描述 |
|------|------|
| **人生危機** | 強制成長的生活事件 |
| **遇到轉化的人** | 遇到能觸發突破的人 |
| **發現熱情興趣** | 發現無法自拔的興趣 |
| **觸底反彈** | 跌到谷底後的反彈 |
| **意外成功** | 開啟新機會的成功 |

### 意圖觸發因素

| 觸發 | 描述 |
|------|------|
| **公開承諾** | 公開責任感 |
| **戲劇性環境改變** | 強制專注的環境變化 |
| **加入密集計畫** | 高強度學習環境 |
| **承擔「不可能的項目」** | 突破舒適區的挑戰 |
| **消除所有分心** | 強制專注的極簡 |

---

## Channel 持續因素

| 因素 | 說明 |
|------|------|
| **清晰願景/使命** | 方向明確 |
| **進步反饋迴路** | 看到進展 |
| **社群支持** | 有人理解 |
| **資源可用性** | 有時間和金錢支持 |
| **健康維護** | 身體撐得住 |
| **壓力管理** | 不會過度焦慮 |
| **整合實踐** | 防止過度專注 |

---

## Channel 退出模式

| 退出信號 | 描述 |
|---------|------|
| **能量逐漸下降** | 專注能量自然消退 |
| **注意力自然轉移** | 興趣開始轉向其他領域 |
| **完成感** | 感覺任務完成 |
| **新問題出現** | 這個問題解決後浮現下一個 |
| **疲憊需要休息** | 身體發出停止信號 |
| **生活要求注意力** | 現實生活開始呼喚 |
| **整合需求出現** | 需要沉澱和消化 |

---

## Channel 持續時間

| 等級 | 常見持續時間 |
|------|-------------|
| **Level 1** | 通常數週到數月 |
| **Level 2** | 數月到數年 |
| **Level 3** | 可能無限期維持高效 Channel |

> Lower levels experience shorter channels due to life problems pulling them out.

---

## Flow 與 Channel 的關係

| 區分 | Flow | Channel |
|------|------|---------|
| **持續時間** | 數分鐘到數小時 | 數月 |
| **身份重組** | 不涉及 | 涉及根本性身份重組 |
| **包含 Flow** | 每次 Flow 是獨立的 | Channel 內含數百個 Flow 事件 |
| **觸發條件** | 挑戰-技能平衡 | 通常需要 crisis 或機會 |

## Arena 中的 Channel 應用

### Stage 1 識別問題

```
「告訴我更多關於這個執念——每天多少小時？」
「什麼觸發了這段 intense 專注期？」
「你犧牲了什麼來維持這個動量？」
```

### Channel 使用策略

| 用戶狀態 | AI 建議 |
|---------|--------|
| **即將進入 Channel** | 提供 Channel 維持因素清單 |
| **正在 Channel 中** | 提醒健康維護和整合實踐 |
| **Channel 即将结束** | 協助識別完成感和下一個問題 |
| **長期停留在 Channel** | 檢查是否忽視其他象限 |

### Channel 後整合

```
整合工作：
1. 記錄什麼有效
2. 識別關鍵原則
3. 建立可持續實踐
4. 與他人分享
5. 為下一個週期做準備
```

## 常見 Channel 模式

| 象限 | Channel 表現 |
|------|-------------|
| **心智** | 沉浸在深度思考中，廢寢忘食，導致突破性框架 |
| **身體** | 發現訓練方法論，3個月超越過去數年 |
| **關係** |  honeymoon phase，哲學轉化，深度連結 |
| **價值** | 最大清晰度，產品發布/商業轉向/創意突破 |

---

## Glitch 與 Channel 的關係

Glitch 是高風險加速器，可能強制進入 Channel：

| Glitch 類型 | 對 Channel 的影響 |
|-------------|-----------------|
| **迷幻藥** | 可能觸發 Spirit Channel，但需要整合 |
| **PEDs** | 可能強制進入 Body Channel |
| **財務壓力** | 可能強制進入 Vocation Channel |
| **極端生活改變** | 可能觸發多象限 Channel |
| **AI** | 可能同時加速所有象限的 Channel |

---

## AI 報告中的 Channel 處理

### 當檢測到 Channel 時的報告段落

```
🎯 Channel 識別

你目前似乎正在經歷 [象限] 的 Channel。
持續時間：[根據用戶描述估算]
進入信號：[觀察到的信號]

Channel 維持策略：
- 確保 [弱勢象限] 不被完全忽視
- 建立整合實踐防止過度專注
- 追蹤 [具體指標] 的變化

退出計劃：
- 識別 Channel 完成的信號
- 準備下一個發展階段的過渡
```
