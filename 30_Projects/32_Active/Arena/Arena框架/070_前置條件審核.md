## Arena 加速器使用前提

> 本章改編自 K-HUMAN3-048_加速器使用前提.md
> Arena 本身就是一個「個人成長加速器」——在使用前，你需要確認自己滿足基本前提。

### 為什麼需要前提？

Arena 不是「報名就能上」的課程。HUMAN 3.0 打了個比方：

> **類固醇**：可以加速肌肉生長，但前提是你的訓練系統和恢復系統要先就位。沒有訓練框架，吃再多類固醇也不會長肌肉。

Arena 的加速效果同理——前提條件沒滿足，你不會進化，只會浪費時間或產生錯覺。

### 七項前提清單

在進入 Arena Stage 1 之前，確認以下七項前提：

| # | 前提 | 核心問題 | 檢測方式 |
|---|------|---------|---------|
| **1** | 最大化自然潛能 | 我是否已經充分發揮了當前天賦？ | 你需要已走過「正常工作 3-5 年」的基礎考驗 |
| **2** | 清晰的北極星 | 我是否清楚知道自己要什麼？ | 沒有方向時優化路線圖，只會讓你更快到達錯誤的地方 |
| **3** | 足夠的當前平衡 | 我的基本生活系統是否穩定？ | 財務/健康/人際沒有急性危機 |
| **4** | 支持系統就位 | 我有沒有人在教我？ | 教練、同儕社群或 AI 反饋機制 |
| **5** | 清晰的進入與退出策略 | 我如何知道 Arena 何時完成使命？ | Stage 4 必須有結束的定義 |
| **6** | 承受不適的能力 | 我願意為成長付出多少不舒服？ | 2.0 以上的進化需要離開舒適圈 |
| **7** | 值得冒險的具體目標 | 我的目標是否足夠具體，值得我全力以赴？ | 「我想成功」不是前提，「我想在 18 個月內建立穩定副業」才是 |

### 等級適用性

| 等級 | Arena 適用性 | 說明 |
|------|-------------|------|
| **1.0** | ⚠️ 需先完成基礎建設 | 1.0 等級需要先建立基本生活結構，再進入 Arena |
| **2.0** | ✅ 完全適用 | Arena 是 2.0 等級的最佳進化工具 |
| **2.5+** | ✅ 適用（需注意加速器悖論） | 有足夠自省能力，但需避免過度依賴 AI（見 [[047_AI輔助風險]]）|

### 前提未滿足的處理方式

| 前提缺口 | 建議行動 |
|---------|---------|
| 基本生活不穩定 | 先用 046 [[046_好時光壞時光日誌]] 建立覺察，等生活穩定後再進入 Arena |
| 沒有北極星 | 先做 045 [[045_多重角色畫布]] 探索方向 |
| 沒有支持系統 | 先加入 Arena 社群或找教練，再開始 Stage 1 |

## 推論邏輯

### 為什麼這些數字是「原創推論」？

前置條件審核的數字沒有隨機對照實驗支持，也沒有橫斷面研究證明「沒有 FQ 2.0 就無法創業」。這些數字是基於以下推論鏈：

```
理論基礎 1：HUMAN 3.0 等級理論
  每個等級代表意識層級的基本特徵
  等級 1.0 = 服從外部規則
  等級 2.0 = 內在驅動追求差異
  等級 3.0 = 多視角整合

理論基礎 2：遊戲化設計原理
  等級 2.0 是大多數「有意義的挑戰」的門檻
  等級 2.5 = 「專業」與「業餘」的分界
  等級 3.0 = 「大師」層級
```

**創業/自由職業 的推論鏈**：

```
為什麼 FQ 2.5？
  創業需要承擔財務風險（FQ 2.0+）
  同時需要主動收入>被動支出（FQ 2.5）
  → FQ 2.5 是「可以負擔創業失敗」的門檻

為什麼 HQ 2.0？
  創業需要持續高能量（90+ 小時/週）
  HQ 1.5 = 普通人的日常能量
  HQ 2.0 = 可以長時間承受壓力的能量系統
  → HQ 2.0 是「創業者體能」的門檻

為什麼 EQ 2.0？
  創業需要領導力（EQ 1.5+）
  同時需要處理團隊衝突（EQ 2.0）
  → EQ 2.0 是「基本領導力」的門檻

為什麼 PQ 2.0？
  創業過程高壓，需要玩商緩衝
  PQ 1.5 = 偶爾玩樂
  PQ 2.0 = 可以把挑戰當遊戲
  → PQ 2.0 是「在高壓下保持玩心」的門檻

為什麼 AQ 2.2？
  創業失敗率 > 90%，逆境商不足會快速burnout
  AQ 2.2 = 可以在逆境中保持方向
  → AQ 2.2 是「持續創業」的韌性門檻

為什麼 XQ 2.2？
  創業者需要方向不斷調整（後設認知）
  XQ 2.0 = 基本自我覺察
  XQ 2.2 = 可以在逆境中仍保持清明
  → XQ 2.2 是「持續修正方向」的覺察門檻
```

**系統化規模化 的推論鏈**：

```
為什麼 FQ 3.0？
  規模化需要被動收入覆蓋固定成本
  這是「財務遊戲設計師」等級
  → FQ 3.0 是「遊戲設計師」門檻

為什麼 EQ 2.5？
  規模化需要管理團隊（不是領導，是管理）
  EQ 2.5 = 情緒智商可以處理系統性衝突
  → EQ 2.5 是「系統管理者」門檻

為什麼 AQ 2.5 / XQ 2.5？
  規模瓶頸是認知和韌性的雙重考驗
  → AQ 2.5 + XQ 2.5 是「持續進化」的門檻
```

> ⚠️ **重要提醒**：這些數字是起點，不是聖經。用戶的獨特情況可能完全偏離這些數字——AI 的職責是根據實際情況調整，而非盲目應用表格。

## 執行順序建議邏輯

### 優先順序原則

1. **先發展差距最小的商數** — 建立信心
2. **再發展對目標影響最大的商數** — 加速前進
3. **同時發展支持性商數** — 建立系統支持

### 範例

```
目標：創業（FQ 需要 2.5 等級）
  ↓
前置條件審核：
  - FQ 需要 2.5（當前 1.8，缺 0.7）
  - HQ 需要 2.0（當前 1.3，缺 0.7）
  - EQ 需要 2.0（當前 1.0，缺 1.0）
  - PQ 需要 2.0（當前 1.0，缺 1.0）
  - AQ 需要 2.2（當前 1.5，缺 0.7）
  - XQ 需要 2.2（當前 1.8，缺 0.4）← 差距最小
  ↓
執行順序：
  1. 先發展 XQ（差距最小，建立信心）
  2. 再發展 FQ（核心目標）
  3. 同時發展 HQ（能量支持）
  4. 最後發展 EQ/PQ/AQ（系統支持）
```

## 與跨商傳導的對應

前置條件審核是 [[060_跨商傳導]] 的實際應用：

- **解鎖順序** — 發展某商數可以為其他商數創造突破口
- **阻礙模式** — 落後的商數會拖慢其他商數的發展

## 使用時機

前置條件審核在 Stage 2 規劃目標時使用：
1. 用戶提出大目標
2. AI 計算該目標需要的維度等級
3. AI 比較用戶當前維度等級
4. AI 給出差距分析與執行順序建議
