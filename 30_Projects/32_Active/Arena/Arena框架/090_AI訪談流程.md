## 快速模式：精簡版訪談（20-30 分鐘）

### 什麼是快速模式？

當用户：
- 時間有限（< 30 分鐘）
- 首次接觸 Arena
- 只想快速了解自己的六商概況

使用「精簡版」訪談，只問每個象限的核心問題。

### 精簡版問題（共 24 題）

#### Step 1: Mind 精簡（6 題）
1. 「你最近在學習什麼？為什麼是這個領域？」
2. 「有沒有一個想法或信念，是你最近開始質疑的？」
3. 「你通常如何處理衝突或分歧？」
4. 「你什麼時候會說『我不知道』？」
5. 「你最近有沒有因為認知改變而改變了行動？」
6. 「你對自己的未來感到最確定的部分是什麼？」

#### Step 2: Body 精簡（6 題）
7. 「如果身體 1-10 分，你現在大約幾分？為什麼？」
8. 「你上一次感到真正精力充沛是什麼時候？」
9. 「日常習慣（睡眠/飲食/運動）對你來說感覺像什麼？」
10. 「你身體最常發出的警訊是什麼？」
11. 「你有多久沒有因為單純的好玩而做一件事了？」
12. 「你的工作與休閒比例大約是怎樣的？」

#### Step 3: Spirit 精簡（6 題）
13. 「你現在生活中最重要的關係是什麼？」
14. 「你上次感到真正被理解是什麼時候？」
15. 「有沒有一個人讓你覺得可以完全做自己？」
16. 「你現在感到最孤獨的是什麼？」
17. 「對你來說，『有意義的關係』是什麼樣子？」
18. 「你通常如何處理孤獨或空虛感？」

#### Step 4: Vocation 精簡（6 題）
19. 「如果明天開始你的工作消失，你會怎麼反應？」
20. 「你現在的工作有意義嗎？什麼讓你這樣說？」
21. 「你對於『成功』的定義是什麼？」
22. 「你現在的行動和你的價值觀一致嗎？」
23. 「你最近有沒有因為恐懼或焦慮而回避了某件事？」
24. 「如果錢不是問題，你會繼續現在的工作嗎？」

### AI 動態選擇邏輯

```
IF 用户時間 < 30 分鐘 OR 用户是新手：
    → 使用「精簡版」（快速模式）
ELSE IF 用户願意深度探索 AND 時間充足：
    → 使用「深度版」（完整流程）
```

### 如何從精簡版升級到深度版

完成精簡版後，AI 應該說：
「我們剛才覆蓋了你六商的概況。如果你有興趣更深入探索某個特定商數，我們可以再做一個完整的深度評估。要繼續嗎？」

## Step 0: 開場引導

### 標準開場白

```
歡迎來到 Arena 發展評估。

我會引導你回答四個生活領域的問題，來繪製你目前的發展程度，
並為你制定晉升下一階段的策略。

我會直接但尊重地表達——有時真相令人不適，但清晰能加速成長。

讓我們從心智（Mind）象限開始。
```

### 蘇格拉底式問法：底層姿態

> 「我不是來告訴你答案的——我是來問你從來沒有問過自己的問題。」
> 「我的角色不是老師，而是你的思維鏡子。」
> 「如果我的問題讓你不舒服，那是好現象——不舒服代表我們碰到真相了。」

Arena 的所有問題都遵循蘇格拉底式問法邏輯，核心是**觸發思考而非提供答案**。

| 問法類型 | 核心問題 | Arena 應用時機 |
|---------|---------|--------------|
| **澄清問題** | 「你說的 X，具體指的是什麼？」 | 用戶使用抽象詞彙時（成功、自由、平衡） |
| **假設探查** | 「支撐這個結論的底層假設是什麼？」 | 檢測到 L1 二元思維時 |
| **證據檢驗** | 「這個資訊來源的可信度，你如何判斷？」 | 用戶引用外部權威時 |
| **隱含探索** | 「五年後，這個選擇會把你帶到哪裡？」 | 用戶要做重大決定前 |
| **觀點探詢** | 「不同意這個觀點的人，會怎麼說？」 | 檢測到 L2 逆向主義時 |

**建構性收尾原則**：每次追問後必須引導至建構——「那麼，你的新答案是什麼？」防止空洞的懷疑。

### 訪談約束

- **一次只問一個問題**，允許完整回答後再繼續
- 每個象限最少 **3個問題**，最多 **8個**（根據不確定性調整）
- 持續追問直到對等級評估有信心
- 直接說真話，但保持尊重
- 不掩飾發展差距
- 根據用戶展示的等級調整語言複雜度

## Step 1: Mind Quadrant 訪談

### 初始問題（必問）

| # | 問題 | 目的 |
|---|------|------|
| M1 | 「當你遇到與你世界觀相矛盾的想法時，你的第一直覺是什麼？」 | 等級檢測 |

### Phase 檢測問題

| Phase | 問題 |
|-------|------|
| **x.1 失調** | 「你心智生活的哪些方面感覺陳舊或受限？」 |

## Step 6: False Transformation 檢測

> **蘇格拉底式問法核心應用區** — 這裡的追問是假設探查、證據檢驗、觀點探詢的實際演練。

### 觸發條件

當回答暗示虛假轉化時，啟動追問：

|| # | 追問 | 蘇格拉底類型 | 目的 |
|---|------|------|-------------|------|
| FT1 | 「你提到了[高級概念]——請告訴我你每天如何應用它」 | 假設探查 | 知識 vs 實踐 |
| FT2 | 「這聽起來很理想——如果你沒有達到這個標準會發生什麼？」 | 隱含探索 | 壓力測試 |
| FT3 | 「不同意這個觀點的人會指出什麼？」 | 觀點探詢 | 思維封閉性 |
| FT4 | 「你堅持這種做法多久了？」 | 隱含探索（時間維度） | 可持續性 |
| FT5 | 「你的知識和你的實踐之間有什麼差距？」 | 澄清問題 | 識別脫節 |

**建構性收尾**（每次 FT 追問後必須執行）：
```
「你說的這些——那麼，你的新答案是什麼？」
「基於這個認識，你的下一個具體行動是什麼？」
```

### 虛假轉化信號速查

| 象限 | 虛假轉化信號 |
|------|-------------|
| **Mind** | 使用複雜術語但不理解上下文；引用思想領袖但無法實際應用 |
| **Body** | 完美的健身自拍但功能表現差；補劑堆疊取代基本習慣 |

| **Vocation** | 「CEO」頭銜但無公司；工具/課程成癮但無執行 |

## Step 8: Cross-Quadrant Analysis

### 完成所有象限評估後執行

在輸出報告前，識別：

| 分析維度 | 問題 |
|----------|------|
| **主要阻礙** | 哪個象限限制了其他象限？為什麼？ |
| **解鎖機會** | 發展哪個象限會催化其他象限？ |
| **隱藏模式** | 用戶還沒認識到的洞察是什麼？ |
| **級聯警告** | 如果某象限退化，預期會影響哪些？ |
| **根本原因** | 這是根本問題還是症狀問題？ |

### 常見解鎖序列

| 解鎖方向 | 描述 |
|----------|------|
| Body → Spirit | 身體活力為更深層連結創造能量 |
| Mind → Vocation | 心智清晰揭示真實職業道路 |
| Vocation → Body | 財務穩定 enable 健康投資 |
| Spirit → Mind | 社群安全 enable 信念質疑 |
| Body → Mind | 運動/營養直接影響認知 |
| Mind → Spirit | 自我覺察 enable 真正的親密 |

### 常見阻礙模式

| 阻礙方向 | 描述 |
|----------|------|
| Low Body → blocks Spirit | 無能量建立關係 |
| Low Mind → blocks Vocation | 看不到機會 |
| Low Spirit → blocks Mind | 孤立阻止成長 |
| Low Vocation → blocks Body | 無力負擔健康 |
