# 073_變現階梯

> Arena 的變現階梯系統 — 融合商業模式畫布
> 產出時間：2026-04-01

---

## 融合說明

Arena 的變現階梯採用商業模式畫布的收入來源區塊，用於 Stage 2-3 的貨幣化路徑設計。

---

## 核心定義

變現是從價值主張到商業模式的閉環。變現階梯描述一人企業從服務到產品的演進路徑。

---

## 三階段變現路徑

### Stage 1：實證服務 (PoC)

**核心**：用技能換取現金，收集痛點與驗證方法論

| 項目 | 內容 |
|------|------|
| **變現方式** | 顧問費、教練費、 Freestyle |
| **時間槓桿** | 低（時間換錢） |
| **驗證目標** | 確認痛點真實、方法論有效 |
| **收入天花板** | 有限（受限於個人時間） |

**目標**：賺到第一筆錢，驗證市場需求

---

### Stage 2：經驗模式化

**核心**：將解決方案流程化、模板化

| 項目 | 內容 |
|------|------|
| **變現方式** | 課程、訓練營、模板、數位產品 |
| **時間槓桿** | 中（一次製作，多次銷售） |
| **驗證目標** | 確認產品可以規模化交付 |
| **收入天花板** | 擴大（但仍需主動交付） |

**目標**：建立可複製的產品，擴大收入

---

### Stage 3：百萬美元產品

**核心**：實現時間槓桿，產品在睡覺時工作

| 項目 | 內容 |
|------|------|
| **變現方式** | 訂閱制、會員制、被動收入、聯盟行銷 |
| **時間槓桿** | 高（系統為你工作） |
| **驗證目標** | 建立持續、被動的收入流 |
| **收入天花板** | 擴大（被動收入） |

**目標**：建立系統，收入與時間脫鉤

---

## Arena Stage 對應

| Arena Stage | 變現階段 | 任務 |
|------------|----------|------|
| Stage 2 | Stage 1-2 | 確認變現方向，開發最小可行產品 |
| Stage 3 | Stage 2-3 | 規模化產品，建立被動收入 |

---

## 一人企業變現方式

| 方式 | 適合階段 | 時間槓桿 |
|------|----------|----------|
| **顧問/教練** | Stage 1 | 低 |
| **課程/訓練** | Stage 2 | 中 |
| **會員制** | Stage 3 | 高 |
| **數位產品** | Stage 2-3 | 高 |
| **聯盟行銷** | Stage 3 | 高 |
| **授權/IP** | Stage 3 | 高 |

---

## 收入組合建議

```
收入組合建議：
- 核心數位產品：50-70%
- 外部槓桿（聯盟行銷）：10-20%
- 高端服務（顧問）：10-20%
```

---

## 變現核心公式

```
變現 = 捕捉注意力 → 轉化為可交付價值
```

```
產品 = 價值（解決方案）
流量 = 受眾（你吸引的人群）
```

---

## AI 訪談應用

當用戶談到「如何賺錢」的問題時：

1. **識別當前階段**：用戶目前在變現階梯的哪個階段？
2. **評估準備度**：用戶是否有驗證過的痛點和方法論？
3. **設計下一步**：從當前階段到下一階段需要什麼？
