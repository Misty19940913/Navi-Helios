# Arena - AI 驅動的人生成長系統

> Arena 是一個以 AI 為核心的人生成長系統，融合設計人生、商業模式、HUMAN 3.0 三大框架，產生可持續、可驗證、可迭代的人生成長框架。

---

## 產品定位

| 項目 | 內容 |
|------|------|
| **產品名稱** | Arena — AI 驅動的人生成長系統 |
| **目標使用者** | 個人（一般大眾） |
| **販售形式** | 課程 |
| **定價** | 之後再定 |
| **使用場景** | 自用、教練對客戶使用 |

---

## 核心問題

> **「如何建立一個可持續、可驗證、可迭代的人生成長系統——在過程中整合意義、價值與所有生活象限，並且隨時知道自己走在正確的方向上。」**

---

## 框架融合方式

Arena 的目標不是「引用三個框架」，而是**蒸餾三個框架的精華，產生第四個新框架**。

```
HUMAN 3.0 = 螺旋動力學 + AQAL + 自我發展九階段 + 心流 + 古代哲學 + 現代職業理論
                ↓ 融合
           新的多維賦能框架

Arena = 設計人生 + 商業模式 + HUMAN 3.0
              ↓ 融合
      新的 AI 驅動人生成長系統
```

### 融合配方

| 源框架 | 提煉 | 融入 Arena |
|--------|------|-----------|
| 設計人生 | 儀表板×設計思考×原型測試 | 現況覺察 × 迭代心態 |
| 商業模式 | 多重角色畫布×價值交換 | 目標萃取 × 路徑規劃 |
| HUMAN 3.0 | 六商評估架構 × 等級進化 × 整合發展觀 | 評估架構 × 整合發展觀 |

### 三系統在 Arena 中的定位

| 系統 | Arena 中的角色 |
|------|--------------|
| **設計人生** | 儀表板（健康、工作、遊戲、愛）— 現況覺察 |
| **HUMAN 3.0** | 四象限（心智、身體、精神、職業）— 意識發展主框架 |
| **商業模式** | 多重角色畫布 — 自我認知**後**的價值交換建模 |

### 二、三等級（意識階段）

每個商數都有三個意識等級，代表個人在該商數的發展層次。

| 等級 | 名稱 | 說明 | 特徵 |
|------|------|------|------|
| **1.0** | 順從者 | 外部驅動，符合期待 | 非黑即白思維、服從權威、視野狹窄 |
| **2.0** | 個人主義者 | 內部驅動，追求差異 | 逆向思考、自信、追求成就 |
| **3.0** | 整合者 | 多視角整合，策略選擇 | 悖論整合、創造新遊戲、系統建構 |

> **重要提示**：你不會「脫離」某個等級——你會超越並融入。較高等級整合較低等級，必要時有意識地運用較低能力。

---

### 三、三階段（發展動態）

每個等級內有三個發展階段，代表當前的發展狀態。

| 階段 | 名稱 | 說明 | 特徵 |
|------|------|------|------|
| **x.1** | 失調 | 舊模式失效 | 痛苦/焦躁、知道要改變但不清清楚 |
| **x.2** | 不確定 | 踏入未知 | 嘗試新方法、脆弱但成長導向 |
| **x.3** | 發現 | 找到有效資源 | 整合新能力、為下一階段做準備 |

---

### 六商×三等級×三階段 評估輸出範例

```
FQ：2.2 — 個人主義者，正在不確定階段
HQ：2.3 — 個人主義者，剛進入發現階段
EQ：1.1 — 順從者，感受到嚴重失調

AQ：2.0 — 個人主義者，穩定狀態
XQ：1.8 — 個人主義者，正在探索階段
```

## 流程系統：四階段框架

```
Stage 1：認識自己
  └─ 現況評估（六商×三等級×三階段）
       ↓
Stage 2：規劃目標
  └─ 大目標建立 → 前置條件審核 → 階段拆解
       ↓
Stage 3：策略規劃
  └─ 路徑設計 → 資源盤點 → 行動計劃
       ↓
Stage 4：迭代執行
  └─ PDCA → 階段調整 → 回 Stage 1
```

### Stage 銜接：硬性門檻

Arena 的 Stage 銜接採用硬性門檻設計——每個 Stage 有最低產出標準，必須達標才能進入下一個 Stage。

**為什麼用硬性門檻？**
- 軟性銜接容易讓用戶跳過關鍵覺察
- 沒有足夠自我認知基礎的目標設定是空中樓閣
- 硬性門檻確保 Stage 1-4 的數據 integrity

**Stage 銜接標準：**

Stage 1 → Stage 2：必須完成
□ 六商現況評估（FQ/HQ/EQ/PQ/AQ/XQ 皆有等級+階段）
□ 生活方式原型識別
□ 至少識別出 1 個 x.1 失調信號

Stage 2 → Stage 3：必須完成
□ 至少 1 個商數達到目標等級
□ 階段拆解覆蓋所有六商
□ 前置條件審核邏輯清晰

Stage 3 → Stage 4：必須完成
□ 30天行動計劃覆蓋所有六商
□ PDCA 追蹤系統建立
□ 至少 1 個商數完成一輪進化評估

### 階段銜接與驗證邏輯

```
Stage 1 產出 → 必須能回答 Stage 2 的核心問題
Stage 2 產出 → 必須能回溯 Stage 1 的自我認知
Stage 2 產出 → 必須能指引 Stage 3 的策略方向
Stage 3 產出 → 必須能對應 Stage 2 的目標設定
Stage 4 產出 → 必須能驗證 Stage 1-3 的所有假設
Stage 4 執行結果 → 回到 Stage 1，重新開始
```

## Stage 2：規劃目標

**問題**：我要去哪？

**核心任務**：
1. **大目標建立** — 你想要什麼？（結合價值主張與等級進化方向）
2. **前置條件審核** — 達成目標需要的商數等級（遊戲邏輯）
3. **差距分析** — 現在 vs. 需要（差距多大？）
4. **階段拆解** — 大目標 → 階段目標
5. **執行順序建議** — 商數發展優先順序

### 前置條件審核機制（遊戲邏輯）

```
目標：抵達商數 X 等級 Y
  ↓
前置條件審核：
  - FQ 需要 2.0 等級
  - HQ 需要 2.0 等級
  - EQ 需要 1.5 等級
  - PQ 需要 1.5 等級
  - AQ 需要 2.2 等級
  - XQ 需要 1.8 等級
  ↓
差距分析：
  - FQ：當前 1.3 → 需要 2.0（缺 0.7）
  - HQ：當前 2.1 → 需要 2.0（超出 0.1）
  - EQ：當前 1.0 → 需要 1.5（缺 0.5）
  - PQ：當前 1.2 → 需要 1.5（缺 0.3）
  - AQ：當前 2.1 → 需要 2.2（缺 0.1）
  - XQ：當前 1.5 → 需要 1.8（缺 0.3）
  ↓
執行順序建議：
  1. 先發展 AQ（差距最小，建立信心）
  2. 再發展 FQ（財務是槓桿）
  3. 同時發展 EQ（支持系統）
  4. 最後發展 HQ/PQ（需要前者的能量支持）
```

**產出**：未來目標（含階段拆解）

## AI 驅動機制

Arena 的 AI 不是回答問題的工具，而是：

1. **自適應訪談** — 根據你的回答調整問題，逐步建構評估
2. **前置條件計算** — 根據目標計算需要的商數等級
3. **差距分析** — 自動計算現在與目標的距離
4. **路徑生成** — 根據差距與優先順序生成行動計劃
5. **迭代引導** — 根據執行結果調整下一輪

> AI 是系統的心臟——持續互動、學習、驅動四階段循環。

## 適用範圍：Arena 能做什麼，不能做什麼

### Arena 的邊界

Arena 是一個**個人成長與職業發展框架**，設計用於：

✅ **Arena 能處理的**
- 職業發展方向不明確
- 工作/生活平衡問題
- 多個人生角色衝突
- 價值觀模糊或動力不足
- 習慣養成與執行力問題
- 人際關係困擾（一般性）
- 創業/變現方向的摸索

❌ **Arena 無法處理的（需要專業介入）**
- 創傷後壓力症（PTSD）、長期憂鬱/焦慮症
- 財務危機（破產、嚴重債務）需要法律或財務專業
- 嚴重成癮行為（藥物、酒精、賭博）
- 飲食障礙（暴食症、厭食症）
- 自殺傾向或嚴重自我傷害
- 家庭/親密關係暴力
- 確診精神疾病的治療

### 危機時期的建議

當你處於急性危機期（財務危機、健康危機、情感危機），**不是**開始 Arena 的好時機。

建議：
1. 先處理危機（找人協助、尋求專業支持）
2. 危機穩定後，再進入 Arena 從 Stage 1 開始
3. 如果你不確定是否適合 Arena，請諮詢心理健康專業人員

### Arena 與專業支援的協作

如果你正在接受心理諮商或治療，Arena 可以作為日常覺察工具配合使用，但：
- 不要用 Arena 取代專業治療
- 如果某個 Arena 練習讓你感到強烈情緒反應，請與你的治療師討論
