# 工業 AI Agent - 概念評估

## 基本資訊

| 欄位 | 內容 |
|------|------|
| 日期 | 2026-04-16 |
| 原始需求 | 有人希望 Boss 運用對 AI 的了解和操作經驗，規劃一個透過 AI 運行大型工程機器的想法 |
| 目前階段 | 概念階段，無詳細資訊 |
| 接洽状態 | 初步接洽，待進一步了解 |

---

## 需求摘要

### 原始描述（轉述）

- 對方希望 Boss 參與一個「透過 AI 運行大型工程機器」的想法
- Boss 認為議題層級過高，實際上是「連接本來就由電腦操作的設備（如車床）」

### 真正問題

```
科技廠需要很多工程師來維持產線正常運作
希望透過 AI 來減輕工程師的負擔
```

### 產品定位

| 項目 | 內容 |
|------|------|
| 產品類型 | 可客製化的 AI Agent（軟體/邊緣設備） |
| 目標客戶 | 工廠、科技廠、製造業 |
| 部署方式 | 邊緣為主（數據保密需求） |
| 收費模式 | 租用（月/年費） |
| 商業機密 | AI 後台設計不可讓客戶得知 |

---

## 技術可行性評估

### 現有技術成熟度（2026-04）

| 領域 | 成熟度 | 說明 |
|------|--------|------|
| AI 視覺辨識 | ✅ 成熟 | 物體偵測、距離判斷、威脅識別 |
| 路徑規劃 | ✅ 成熟 | 自動導引車（AGV）在工廠已運行多年 |
| 遙控操作 | ✅ 成熟 | 危險環境的遠端控制已有實例 |
| AI 異常偵測 | ✅ 成熟 | 設備監控、預測性維護 |
| 完全自動駕駛 | ⚠️ 受限 | 複雜環境仍需人類判斷 |

**結論：技術上可行，但非完全自動化，而是「AI 輔助人類」模式**

## 技術需求分析

### 設備接口挑戰

| 挑戰 | 說明 |
|------|------|
| 多廠牌設備 | 不同設備有不同接口（OPC-UA / Modbus / 專有協議）|
| 標準化問題 | 需要一個標準化適配層 |
| 複雜度 | 這是一個大工程 |

### 邊緣部署需求

| 需求 | 解法 |
|------|------|
| 數據保密 | 邊緣計算，資料不外流工廠 |
| 即時性 | 邊緣設備具備足夠算力 |
| 離線運作 | AI Box 獨立運作 |

### 商業模式對應

| 商業需求 | 技術解法 |
|---------|---------|
| 租用而非販售 | AI Box 所有權在產品方，客戶只有使用權 |
| 保護 AI 機密 | 模型加密、設備不對外暴露 |
| 多廠牌設備 | Interface Adapter 層（可插拔）|
| 持續更新維護 | 遠端加密更新通道 |

## 已知限制

| 項目 | 狀態 |
|------|------|
| 沒有具體產業 | 未確認是半導體/CNC/傳統製造 |
| 沒有具體設備 | 未確認目標設備類型 |
| 沒有詳細需求 | 僅有概念，無技術規格 |
| Boss 非該領域專業 | 僅能評估一般性可行性 |

## 結論

**可行，但需要更多資訊才能進一步規劃。**

這是一個「AI 監控和異常診斷」的產品方向，不是「完全自動駕駛工程車」。

MVP 可從「單一設備類型」切入，驗證後再擴展。
