# kb-curator 技能

此技能旨在維護 Navi Helios 系統中 `400知識/410原子化知識庫` 的長效秩序與原子化純度。

## 核心操作流程

### 1. 金庫審核 (Audit)
- **重複清查**：識別語義或名稱重複的檔案（如：括號缺失、英文翻譯有無、標點符號不一致）。
- **YAML 漂移檢測**：識別不符合 `YAML核心規範協議.md` 的檔案，特別是缺失核心 7 行屬性、標籤錯誤或 status 標記不當。

### 2. 標準化校準 (Calibration)
- **命名規範化**：統一採用 `中文標題 (英文翻譯).md`。括號前必須有一個半形空格。
- **YAML 強制對齊**：
    - 將 `status: DONE` 自動映射為 `status: evergreen`。
    - 確保 `subject`, `parent`, `related` 的 Wikilinks 均被 `"` 雙引號包圍。
    - 若 `time_create` 缺失，根據作業系統檔案建立日期補齊。
- **狀態判定邏輯**：
    1. 內容完整且具備實踐案例 -> `evergreen`
    2. 內容具備結構但尚待擴充 -> `in-progress`
    3. 僅有初步摘要或來源連結 -> `seed`

### 3. 合併與去重 (Consolidation)
- **內容整合原則**：將舊版詳細筆記與新版原子卡片合併時，保留舊版的「實踐指南/案例」並採用新版的「核心洞見 CALLOUT」。
- **刪除舊檔**：完成合併並確認內容無誤後，移除重複檔案。

### 4. 連結修復 (Link Remediation)
- 當檔案更名或合併後，必須全局搜尋並更新 Vault 中所有指向該檔案的 `[[Wikilinks]]`。

## 指令觸發建議
- `//CURATE_KB`：對知識庫執行完整審核並產出待處理清單。
- `//FIX_YAML`：對目標目錄下所有筆記執行 YAML 標籤與格式校準。
- `//MERGE_KB [核心關鍵字]`：針對特定主題的重複檔案執行內容併購。

## 輸出規則
- 任何「刪除」或「大量搬移」動作前，必須產出 `📋 待處理變更報告` 並請求 Misty 顯式批准。
- 每次整理結束後，產出當次知識增量摘要至 `600支援/620對話日誌/`。
