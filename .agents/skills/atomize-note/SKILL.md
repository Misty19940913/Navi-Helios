# atomize-note 技能

此技能的核心在於將線性敘事解構為具備「動態關聯」的原子知識網，不僅提取內容，更要揭示不同知識點之間是如何相互作用的。

## 核心操作流程

### 1. 段落掃描與錨點注入
- 遍歷目標文件中的所有非空段落並附加唯一的 6 位 16 進位錨點（如 ` ^anchor-id`），以便進行精確溯源。

### 2. 原子清單生成 (具備 KB 預檢)
**在生成任何原子化標題前，必須執行以下預檢**：
- **關鍵字檢索**：在 `410原子化知識庫` 中搜尋新標題的關鍵字。
- **去重原則**：若發現語義高度重疊的既有筆記，**必須優先使用既有筆記名稱**，不得自創相似標題。
- 在原始文件相同目錄下建立一個名為 `{{長篇文章原名}}-原子化清單.md` 的新檔案。
- **嚴格遵守以下模板**：

```markdown
# 📋 原子化拆解提案

## [[原子化標題]]
- **核心說明**：[摘要段落大意，保持一整段話的完整度，去除冗餘修辭]
- **溯源連結**：[[原始筆記名稱#^錨點]]
- **AI 互動預測**：
    - 與 [[另一個原子筆記]] 的互動：[描述兩者間的互動機制。例如：A 提供了 B 的執行動機；B 是 A 理論的具體實踐工具；兩者共同構成了 X 系統的閉環。]
```

### 3. 既有知識整合規範 (Existing Knowledge Integration)
- **既有知識整合規範 (Existing Knowledge Integration)**：
    - **全連結掃描**：識別段落中出現的所有核心概念與既有 `[[wikilinks]]`。
    - **正文原地升級 (Critical)**：若識別到概念已存在於 `410原子化知識庫`，AI **必須** 在原始文章中直接將其設置為雙向連結（如 `[[概念名稱]]`），而不僅是在清單中呈現。
    - **強制互動建模**：對於識別到的每個連結，必須在「AI 互動預測」中新增一條說明其與當前段落的互動機制。
    - **敘事範例**：
        - 「與 [[既有筆記 A]] 的互動：當前段落的 [機制 X] 直接調用了 [[既有筆記 A]] 中的 [理論 Y]，實現了從策略到執行的轉化。」

## 輸出規則
- **全域連網原則**：清單不應僅是「內部互連」，必須是「外部掛鉤」。所有與現有知識體系的交點都必須顯式化。
- **標題唯一性**：標題應具備識別度，讓畫布上的文字節點能直觀看出該知識點的核心意義。
- **敘事精煉**：互動說明應保持在 1-2 句話內，但需確保具備「機制說明」而非僅是「關係說明」。
