### **交易記錄與日誌整合方案：量子態雙向同步系統**  
---
#### **Ⅰ. 核心整合邏輯**  
**1. 量子糾纏架構**  
- **單筆交易** → **多日誌呈現**（主日誌+專項報告）  
- **雙向引用**：日誌中顯示交易摘要，交易記錄反向連結至相關日誌  

**2. 時空連續性原則**  
- 交易自動歸檔至`{{date}}`日誌  
- 跨日交易（如分期付款）建立時間錨點  

---

#### **Ⅱ. 實戰配置步驟**  

##### **1. 建立量子糾纏模板**  
```markdown  
---  
type: 量子交易  
date: {{DATE}}  
amount: {{AMOUNT}}  
payee: {{PAYEE}}  
accounts: [{{ACCOUNT}}]  
cross-logs: []  
---  
```  

##### **2. 日誌自動嵌入腳本**  
```javascript  
// 在每日日記模板中加入  
<%*  
const transactions = dv.pages('"財務裝甲/交易記錄"')  
  .where(t => t.date.toFormat("yyyy-MM-dd") === tp.date.now("yyyy-MM-dd"));  

if (transactions.length > 0) {  
  tR += `## 💸 今日財務量子態\n`;  
  tR += dv.table(["金額", "類別", "帳戶"],  
    transactions.map(t => [  
      `$${t.amount}`,   
      t.category,   
      t.account  
    ])  
  );  
}  
%>  
```  

##### **3. 跨日誌連結協議**  
```markdown  
```javascript  
// 在交易記錄模板中  
const relatedLogs = ["晨間檢視", "週末檢討"].map(log => `[[${log}]]`);  
tR += `\n關聯日誌：${relatedLogs.join(" ")}\n`;  
```

#### **Ⅲ. 進階同步功能**  

##### **1. 財務時光機**  
```dataviewjs  
const timeline = dv.pages('"財務裝甲/交易記錄"')  
  .sort(t => t.date)  
  .map(t => `[${t.date.toFormat("MM-dd")}] ${t.payee}: $${t.amount}`);  

dv.paragraph(`### 🕰️ 財務軌跡\n${timeline.join("\n")}`);  
```  

##### **2. 情緒消費分析**  
```dataviewjs  
const moodSpending = dv.pages('"日誌"')  
  .where(p => p.mood && p.財務支出)  
  .groupBy(p => p.mood)  
  .map(g => ({  
    mood: g.key,  
    total: g.rows.sum(p => p.財務支出)  
  }));  

dv.table(["心情", "消費額"], moodSpending);  
```
