# Document Consistency Check（立場 vs 行動交叉檢查）

## 核心問題

同一份文件可能同時包含：
1. **立場段落**（position statement）——「X是Y，不是Z」「我們的原則是...」
2. **具體行動指令**（action items）——「建立X」「導入Y」「設計Z」

兩者可能表面一致但實質矛盾。分析時只讀立場段落、遇到立場文字就終止懷疑，會漏報這種「隱性矛盾」。

---

## 觸發條件

**文件滿足以下任一條件時，強制執行交叉檢查：**

- 包含「立場宣稱」段落（核心價值、核心立場、本質上...、...是...不是...）
- 同時包含具體行動指令（任務清單、待辦、建立、導入、開發、設計、執行）
- 章節結構為「原則層 + 執行層」的分層文件（計劃文檔、系統設計、架構文檔）

**觸發信號**：「確認這份文件的一致性」「有沒有矛盾」

---

## 操作方法（4步交叉檢查）

### Step 1：讀立場段落

識別並記錄所有立場段落的核心主張。

### Step 2：列舉具體行動指令

識別文件中**所有**具體行動指令，以動詞為標誌：
- 建立、創建、開發
- 導入、遷移、整合
- 設計、構建、實作
- 執行、落實、完成

### Step 3：逐條對照

每條行動指令是否與立場段落一致？口徑相同為 ✅，矛盾為 ❌，無法判斷為 ⚠️。

### Step 4：矛盾時回報並停止

發現矛盾時：
- **回報矛盾**（具體指出：立場段落說X，行動段落說Y）
- **不要猜測意圖**
- **不要假設是筆誤或可以「理解為一致」**

## 真實案例（本 session）

**檔案**：`task_plan.md`

**立場段落（Line 119）**：
> 核心立場：從零開始**設計新架構**，現有 vault 是「素材參考」不是「遷移目標」。

**具體行動指令（第五章 T4.x/T5.x）**：
- T4.1：在 `30_Projects` **導入** Intensity 標籤
- T4.2：**建立** `10_Inbox` 的自動化分流 SOP
- T4.3：**建立** `Library_MOC`
- T5.1：**建立** MOC 建立的 SOP

**矛盾分析**：
| 立場 | 行動 | 一致？ |
|------|------|--------|
| vault 是素材參考 | T4.x/T5.x 全部是「建立/導入」（實作） | ❌ 矛盾 |
| 從零設計新架構 | 沒有任何「研究」或「規格」字樣 | ❌ 矛盾 |

**初始錯誤回報**：「文件一致」
**正確回報**：「立場段落宣稱 vault 為素材參考，但第五章全部任務說明使用實作語言（建立/導入/設計），兩者矛盾」

---

## 為什麼會失敗（「只看表面」的根因）

**表面症狀**：立場段落說「vault 是素材」，第五章抬頭說「任務清單」，直覺上「素材 + 任務 = 沒問題」。

**實際原因**：
- 立場段落的口徑是「從零研究/設計」
- 行動段落的口徑是「建立/導入」（實作）
- 「研究/設計」≠「建立/導入」——這是實質矛盾，不是表面用詞差異

**觸發條件缺失**：沒有「立場 + 行動」的交叉檢查触发提前終止。

**本 session 新增教訓**：交叉檢查時列舉 action items 必須**完整**——遇到第一個矛盾就回報是對的，但**列舉所有 action items** 的過程不可中斷。不可只檢查第一條就聲稱「發現矛盾並停止」，必須報告「發現矛盾，同時確認以下 N 條均未檢查」。

本 session 案例：`task_plan.md` 立場段落口徑與第五章不符。檢查時發現 `T4.1` 「導入 Intensity 標籤」與 vault 素材立場矛盾 → 回報。完整列舉後發現 T4.x/T5.x/T6.x 全部 8+ 條均為實作語境，非研究語境。changelog 記錄時又因 `||` 欄位前綴和 python `repr()` 殘留字元導致文件損壞，緊急還原後重新執行。

---

## 常見陷阱

1. **立場段落口徑 vs 行動段落口徑**：立場說「研究」，行動說「建立」→ 實質矛盾
2. **編號跳號**（如 2.3 刪除後殘留 2.4）：暗示資訊遺漏或重構未完成
3. **口語對比不是矛盾**：「不是如何建構系統」vs「研究系統」為語意對比，changelog 本身記錄修正動作不視為矛盾
4. **圖 vs 文不一致**：口述定義（執行鏈 1.2）與圖示（1.1 三系統一體）可能矛盾，需交叉核對

## 修正確認方法

修改後，檢查文件中是否還殘留「建構/實作/導入」等舊語境：
```python
bad_phrases = ['建構', '實作', '設計新架構', '從零開始設計']
for phrase in bad_phrases:
    # 忽略 changelog 本身（記錄修正動作的條目）
    # 忽略「不是如何建構系統」的語意對比句
```

發現殘留 → 確認上下文是否為「修正記錄本身」或「口語對比」，不是新產生的矛盾。
