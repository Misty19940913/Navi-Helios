# 自媒體內容策略系統

## 觸發條件

當用戶提到以下需求時啟動：
- Threads / Instagram 策略
- 自媒體內容規劃
- 變現路徑設計
- 品牌帳號定位
- 「粉鳥鴿 @sentrypigeon」的內容系統

## 核心框架：四層漏斗

```
Threads（鉤子 + 市場測試）
    ↓  「bio 有完整版」
IG（深度教學 + 信任轉換）
    ↓  「加入私域社群，獲得完整模板」
Email / 私域社群（變現基地）
    ↓
付費產品（數位模板 / 訓練營 / 社群 / 諮詢）
```

## 平台分工矩陣

| 平台 | 階段 | 形式 | 核心任務 | 引導動作 |
|:---|:---|:---|:---|:---|
| **Threads** | 測試（Test） | 短文字鉤子 | 驗證選題共鳴度，收集用戶痛點 | 文末：「有興趣的人，bio 有完整版」 |
| **Instagram** | 展示（Display） | 多頁圖文輪播 | 同一主題的完整教學，建立專業權威 | bio 連結導流至私域社群 |
| **Podcast** | 深度（Deep） | 純音頻 | 人性故事 + 深度陪伴，強化黏著度 | 引導加入 Email 訂閱 |
| **YouTube** | 權威（Authority） | 影片 | 系統化教學與長篇案例拆解，爭取長尾流量 | 影片說明欄放私域入口 |

## Threads 執行策略

```
頻率：每天 1-2 篇（不可斷，演算法害怕沉默）

內容比例：
50% — 核心話題（AI工具 × 知識管理 × 非工程人視角）
30% — 互動型（問問題、投票、填空句）
20% — 個人視角（刑警背景、學習歷程、試錯經驗）

禁止：
- 外部連結（Threads 不愛，統一導至 IG bio）
- Topic 飄移（每篇只有一個觀點）
```

## IG 執行策略

```
格式：Carousel 多圖輪播（標題 + 結構 + 截圖 + 結論）
定位：不做「Threads 詳細版」，做「同一主題的完整教學」
目的：把「有興趣的人」轉換為「願意追蹤的人」
```

## 第三層變現（Monetization Layer）

### 收入來源優先順序

| 優先 | 來源 | 形式 | 說明 |
|:---:|:---|:---|:---|
| 1 | **數位模板包** | Obsidian Workflow 模板 | 最小可行產品，最快驗證變現假設 |
| 2 | **訓練營** | 直播＋模板＋社群督促 | 框架＋陪伴，模仿雷蒙三十模式 |
| 3 | **付費社群** | 月費制私域群組 | 持續價值輸出，穩定訂閱收入 |
| 4 | **一對一諮詢** | 單次服務 | 最高單價，時間換錢 |

### 參考案例：雷蒙三十

- **關鍵數據**：IG 不到 4,000 粉絲，電子報 6,500 人 → 年營收 960 萬
- **核心洞察**：粉絲數不是目標，願意付費的鐵粉數量才是
- **變現結構**：數位產品（38%）＋ 版稅（36%）＋ 社群會員（11%）＋ 服務（8.4%）＋ 業配（5%）＋ 聯盟（1.4%）
- **原則**：做社群（Community）不是社群媒體（Social Media）

### 起步門檻

> 不需要粉絲破萬才開放。**100 個願意跟的人**就可以開始第一批早鳥私域。

## 帳號定位：@sentrypigeon 粉鳥鴿

### 差異化核心

```
刑警出身 × 非工程人視角
```

坦誠揭露失敗是最高級的權威。不是工程相關出身 × 直接用 AI 工具的坦誠人設，在中文 AI 創作者圈幾乎沒有直接競爭者。

### 當前帳號問題

| 問題 | 說明 |
|---|---|
| **Topic 飄移** | 同時在聊 Obsidian vault、AI agent 底層原理、刑警日常、就業觀察，演算法無法定型 |
| **深度 vs 篇幅矛盾** | Threads 是短文平台，核心內容需要深度，取捨策略未確立 |
| **曝光靠互推** | 內容有價值但主動 discover 不足 |
| **沒有固定專欄格式** | 演算法偏好的「每週幾固定發 X 類內容」還沒形成 |

### 品牌哲學對照

- **實戰真理觀** → Threads 上「坦誠試錯」人設
- **防禦哲學** → 內容中「不追求捷徑，踏實累積」的框架
- **自我主導** → 讀者看完知道「我可以自己做到」而非「我需要仰賴誰」

| 工具 | 適合度 | 理由 |
|---|---|---|
| Obsidian + 發布插件 | ⭐⭐⭐ | 載具統一，不需要多學工具 |

## 文件更新順序

當用戶要求更新內容系統文件時，**順序如下**：

```
1. `513平台發布策略.md` — 先更新漏斗邏輯 + 第三層變現說明
2. `511品牌核心價值體系.md` — 再更新商業轉化 + 人格一致性
3. `512內容視覺規範.md` — 等視覺素材確認後最後更新
```

> ⚠️ 這個順序是共識，不要顛倒。用戶說「逐一完成」。

## 執行原則

1. **Threads 是市場測試**：先驗證選題是否有共鳴，再投入 IG 深度製作
2. **Threads 不放外部連結**：所有引導匯集在 IG bio
3. **IG 是信任轉換**：每一篇都是獨立的完整教學，不依賴 Threads 上下文
4. **私域很早就開始**：100 個願意跟的人就可以開放第一批早鳥
5. **所有平台最終引導至同一處**：IG bio 是流量中樞
