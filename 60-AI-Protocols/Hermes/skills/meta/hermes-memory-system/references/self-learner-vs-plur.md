# Self-Learner vs PLUR 功能對比

**產出時間：** 2026-05-08
**起因：** 記憶疊加路线图 Step 4 驗證

---

## 核心功能對比

| 維度 | Self-Learner（修復後）| PLUR |
|------|----------------------|------|
| 觸發機制 | Keyword matching（意圖信號偵測）| 用戶明確反饋寫入 YAML |
| 寫入格式 | `.learnings/debug-*.md` | `~/.plur/memory/*.yaml` |
| 假陽性率 | ⚠️ 系統性偏高（2,425 個檔中 98% 為假陽性）| ✅ 結構化指紋，大幅降低 |
| 多 Agent 共享 | ❌ | ✅ |
| 整合层级 | 內建 plugin | 插件系統 |
| 目前狀態 | 修復後待驗證 | 未測試 |

---

## 關鍵差異

**Self-Learner：** 意圖驅動——先猜測用户意圖，再產生輸出，最後過濾。過濾層薄弱導致假陽性堆積。

**PLUR：** 反饋驅動——只接收用户明確糾正，不自己推理對錯。從一開始就杜絕了偵測失敗的問題。

---

## 建議（2026-05-08）

1. 若 self_learner 修復後能在 2 週內穩定（不新增大量假陽性）→ 繼續使用，Step 4（plur）跳過
2. 若 self_learner 再次失效 → 果斷遷移到 PLUR

**PLUR 尚未在此實例上測試，設定前需先研究。**
