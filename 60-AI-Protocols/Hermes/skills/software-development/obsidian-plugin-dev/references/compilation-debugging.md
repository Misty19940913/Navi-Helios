# Obsidian Plugin Compilation Debugging Reference

> Condensed from `obsidian-plugin-compilation-debugging` — getting a plugin skeleton from repo to successful build.

## Core Principle
**永遠先讀 CONTRIBUTING.md**，再執行任何編譯動作。

## 常見 CONTRIBUTING.md 陷阱

| Plugin | 陷阱 |
|--------|------|
| full-calendar | `main.css` 必須 symlink 到 `styles.css`，esbuild 輸出 `main.css`，但 Obsidian 只認 `styles.css` |
| full-calendar | 需安裝 `npm install -g livecycle` 或 hot-reload plugin 加速開發 |

## esbuild.config.mjs 完整範本

```js
import esbuild from "esbuild";
import process from "process";
import builtins from "builtin-modules";

const banner = `/*
THIS IS A GENERATED/BUNDLED FILE BY ESBUILD
if you want to view the source, please visit the github repository of this plugin
*/`;

const prod = process.argv[2] === "production";

esbuild
  .build({
    banner: { js: banner },
    entryPoints: ["src/main.ts"],
    bundle: true,
    external: [
      "obsidian",
      "electron",
      "@codemirror/autocomplete",
      "@codemirror/closebrackets",
      "@codemirror/collab",
      "@codemirror/commands",
      "@codemirror/comment",
      "@codemirror/fold",
      "@codemirror/gutter",
      "@codemirror/highlight",
      "@codemirror/history",
      "@codemirror/language",
      "@codemirror/lint",
      "@codemirror/matchbrackets",
      "@codemirror/panel",
      "@codemirror/rangeset",
      "@codemirror/rectangular-selection",
      "@codemirror/search",
      "@codemirror/state",
      "@codemirror/stream-parser",
      "@codemirror/text",
      "@codemirror/tooltip",
      "@codemirror/view",
      ...builtins,  // node 内置模組
    ],
    format: "cjs",
    watch: !prod,
    target: "es2016",
    logLevel: "info",
    sourcemap: prod ? false : "inline",
    treeShaking: true,
    outfile: "main.js",
  })
  .catch(() => process.exit(1));
```

## 常見缺失

| 問題 | 修復 |
|------|------|
| 缺少 banner | 加入 generated file 警告 |
| 缺少 `obsidian` in external | 執行時會 "Could not resolve 'obsidian'" |
| 缺少 `electron` in external | Electron 環境會爆 |
| 缺少 `builtin-modules` | 需 `npm i -D builtin-modules` |
| 缺少 `format: "cjs"` | Obsidian 只認 CommonJS |
| `target` 太新（如 `es2020`） | 改 `es2016` 確保最大相容性 |

## watch 模式陷阱

```js
// ❌ 錯誤
watch: !prod, // 不能放在 .build() 裡

// ✅ 正確
if (!prod) {
  const ctx = await esbuild.context(options);
  await ctx.watch();
} else {
  await esbuild.build(options);
}
```

## tsconfig.json 檢查清單

```json
{
  "compilerOptions": {
    "target": "es2016",
    "module": "commonjs",
    "strict": true,
    "esModuleInterop": true,
    "skipLibCheck": true,
    "forceConsistentCasingInFileNames": true,
    "outDir": "./dist",
    "rootDir": "./src",
    // ⚠️ 不要有 "noEmit": true，會導致 esbuild 收不到輸出
    "declaration": false
  },
  "include": ["src/**/*"],
  "exclude": ["node_modules"]
}
```

## 常見 TypeScript 錯誤修復

```
TS2307: Cannot find module './settings'
TS2304: Cannot find name 'DEFAULT_LIFEOS_SETTINGS'
TS2339: 'containerEl' does not exist on type '...'
```

修復方式：
- 確認每個 export 的檔案都有對應的 import
- `DEFAULT_LIFEOS_SETTINGS` 這類常量需要從 types.ts 或 settings.ts export
- `containerEl` 是來自 `PluginSettingTab` 的 inherited 成員，子類直接用 `this.containerEl`

## Type 型別不相容

```ts
// 加 as 強制轉型
const cache = app.metadataCache.getFileCache(file) as any;
// 或
const cache = app.metadataCache.getFileCache(file) as CachedMetadata;
```

## full-calendar 專屬陷阱

`main.css` vs `styles.css`：esbuild 輸出 `main.css`，但 Obsidian 只認 `styles.css`。
需手動建立符號連結：
```bash
ln -sf main.css styles.css
```
