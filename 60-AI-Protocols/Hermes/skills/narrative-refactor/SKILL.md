# 敘事極致化重構技能 (Narrative Refactor)

## 觸發條件（何時用）

- 用戶輸入 `//tweak` 或要求「重寫/優化」時啟動
- 「自媒體審計協議」評分低於 7 分時自動啟動
- 用戶輸入 `//rb` 要求套用模板重構筆記時啟動

---

## 執行步驟

### 第一階段：病灶讀取

1. 讀取審計結果中評分最低的維度
2. 識別該維度失敗的根本原因（是開場無力？視角錯誤？缺乏場景？結論太滿？人格模糊？）
3. 確認用戶是否要求 A版（挑戰型）或 B版（友好型），若未指定則同時輸出雙版

---

### 第二階段：五大核心手術模組

#### 1. 開場優化 (Hook Calibration)
- **刪除所有結論**，不給答案
- 只給**懸念或反直覺質疑**，創造「接話空間」
- 範例：將「我認為這個策略有效」改為「你每次都在同樣的地方止損——有沒有想過為什麼？」

#### 2. 視角轉向 (Perspective Shift)
- **第一人稱 → 第二人稱**
- 刪除「我認為」「我觉得」「我相信」
- 改為「你正在經歷……」「你注意到……」「你的手指停在……」

#### 3. 場景轉化 (Atmospheric Transformation)
- 從**瑣碎/極小瞬間**切入
  - 例如：一杯冷掉的咖啡、清晨 3 點的 K 棒、菸灰缸裡的菸蒂
- **禁止在場景中講道理**，讓成像感自帶邏輯
- 讓讀者「看到」而非「讀到」

#### 4. 內容留白 (Strategic Blankness)
- 在**邏輯高潮處斷開**，不給結論
- 結論**留給留言區**
- 信念：文章只是討論的容器，不是終點

#### 5. 人格濾鏡 (Persona Filter)
- 選擇一種冷峻冷色調的人格：
  - **刑警**：銳利、直接戳穿幻象
  - **交易者**：冷靜計算、不帶情緒
  - **資深觀察者**：見過太多，不驚不嘆
- 整篇敘事統一該人格的說話語氣

## 雙版輸出格式

### A版：挑戰型（Challenger）

| 面向 | 風格要求 |
|------|----------|
| 語氣 | 犀利、斷言式、具有主權感 |
| 矛盾處理 | 主動點出邏輯矛盾，不回避 |
| 解方提供 | **不給解方**，只拋問題 |
| 姿態 | 「我知道你錯在哪裡，但我不會告訴你怎麼改」 |
| 結尾 | 以詰問收尾，挑釁讀者思考 |

**適用場景**：高自尊讀者、反權威群體、交易員群體

---

### B版：友好型（Friendly）

| 面向 | 風格要求 |
|------|----------|
| 語氣 | 謙卑但具備洞察、平等對話 |
| 矛盾處理 | 溫和揭示，邀請而非逼迫 |
| 解方提供 | 保留不確定感，「或許……」「說不定……」 |
| 姿態 | 冷靜的同行者，陪你走過那一段 |
| 結尾 | 開放式邀請，而非斷言 |

**適用場景**：新手讀者、需要陪伴感的受眾、溫和改變者

---

## 結尾：勤務挑戰 (Today 1-step)

每篇重構結尾**必須**附帶一個具體、可執行的今日單步行動：

```
🚨 今日勤務挑戰（Today 1-step）：
[具體動作] + [量化標準] + [時間期限]

例如：
把這篇文章的開場口語唸一遍（刻意停在你以前會說結論的地方）。
今天有效。
```

---

## 陷阱/注意事項

| 陷阱 | 說明 |
|------|------|
| 過度留白 | 留白是手段不是目的，全篇懸崖峭壁會讓讀者放棄 |
| 人格分裂 | 五個模組必須統一在一個人格下，切換會破壞張力 |
| A版太囉嗦 | 挑戰型不是髒話連篇，是「一句話刺入核心」 |
| B版太溫吞 | 友好型不是無力，是「有溫度的洞察」，別變成心靈雞湯 |
| 結論殘留 | 手術後仍殘留結論片段，記得用「刪除線」或「飄動問題」取代 |
| 場景冗長 | 場景是鈔票上的水印，不是鈔票本身，點到即止 |
| 雙版混淆 | A版和B版是獨立的敘事策略，不要在同一篇裡切換 |

---

**核心信念**：好的敘事不是告訴讀者答案，而是讓他們在閱讀中自己走到答案門口，發現門沒鎖。
