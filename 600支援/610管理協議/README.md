# 系統指南

本工作區以 Markdown 檔案為主，作為知識管理與個人生產力系統。

## 架構

Navi Helios 3.5 系統

### 000捕獲（收件匣）：
- 功能：存放靈感與資訊碎片。
- 標籤：`#系統/捕獲`
- 規則：實施「7 天處置規則」，逾期未處理（丟棄、轉任務、轉知識）則直接刪除。

### 100夢想（中央指揮部）：
- 功能：定義願景、12週戰略與殘酷裁決（Kill List）。
- 標籤：`#系統/夢想`
- 各領域如需參考長期願景，使用 `![[100夢想/五年願景]]` 或 `[[100夢想/五年願景]]` 方式引用，避免重複維護多份內容，保持單一資料來源

### 200週期紀錄：
- 功能：追蹤長期抱負的進度。
- `210日誌/`：每日戰場。記錄日常任務、項目拆解的行動任務、臨時代辦事項。
  - 結構建議：今日成果、情緒評估、明日重點
  - 標籤：`#系統/週期`
- `220覆盤/`：每周、月、季、年的覆盤紀錄存放區。
  - 標籤：`#系統/週期`
- `230任務池/`：**系統唯一原子化任務存放區**。
  - 功能：存放所有原子化行動任務筆記
  - 標籤：`#系統/任務`
  - 所有任務必須具備 `parent` 屬性並向上溯源至專案或夢想

### 300專案（執行實體）：
- 功能：定義與追蹤重要目標。包含生活中不同責任領域相關的專案（例如：財務、健康、職涯），所有專案必須連結至 100夢想 層級。
- 建議內容：目標描述、期限、里程碑
- 標籤：`#系統/專案`
- 所有專案必須具備 `parent` 屬性並向上溯源至夢想

### 400知識（永久且有組織的知識庫）：
- 功能：存放已整理的知識、模板與工具。
- `410原子化知識庫/`：存放著經過提煉的原子化筆記，以供長期參考和連結。
- `420書籍資料/`：存放書籍的閱讀筆記。
- `430網頁資料/`：存放擷取來自網路文章的筆記。
- `440影片紀錄/`：存放觀影心得的位置。
- `450課程資料/`：存放參與課程的資料位置。
- 標籤：`#系統/知識`
- 所有知識筆記必須具備 `subject` 屬性，指向特定的知識範疇 MOC

### 500自媒體創作區：
- 功能：所取得之知識、資訊轉化為社群內容。內容創作與 KPI 數據。連通 `Home.md` 儀表板。
- 結構：Ideas → Drafts → Schedule → 發佈
- 標籤：`#系統/創作`

### 600支援：
- 功能：底層模板、元件與說明文件。存放已整理的模板、元件、規範與工作流程。
- `610管理協議/`：存放管理協議文件。
- `620Template/`：用於標準化筆記（如：專案、讀書筆記、會議記錄）的範本。
- `630components/`：驅動 `Home.md` 儀表板插件的設定檔。
- `640excaildraw/`：存放 excaildraw 檔案。
- 標籤：`#系統/資源`

### 700存檔（封存）：
- 功能：封存已完成或非作用中項目。存放已完成、過時或不再活躍的專案、任務與筆記。
- 標籤：`#系統/存檔`

## 核心主權原則

1. **權力下行**：所有行動（任務/專案）必須具備 `parent` 屬性並向上溯源。
2. **知識範疇**：所有筆記必須具備 `subject` 屬性，指向特定的知識範疇 MOC。
3. **YAML 最小化**：嚴格遵守核心屬性，確保資料純淨度。系統採用「7 行核心屬性」為底座。

## 命名規則

- 資料夾以三位數字開頭，代表分類與順序，便於未來擴充與排序。
- 使用正體中文或英文
- 建議格式：「索引號-主題-版本」命名
- 索引號：如`300專案/231偵查`
- 版本號：00、01、02 依序遞增

## 知識流動

系統的知識流動遵循以下路徑：

1. **靈感收集**：`000捕獲/` → 整理分類
2. **願景定義**：`100夢想/` → 戰略規劃
3. **任務執行**：`200週期紀錄/230任務池/` → 行動追蹤
4. **專案執行**：`300專案/` → 知識萃取
5. **知識歸檔**：`400知識/` → 永久儲存
6. **內容創作**：`500自媒體創作區/` → 發佈輸出
7. **系統支援**：`600支援/` → 模板與規範
8. **封存管理**：`700存檔/` → 已完成項目

## 檔案命名

- 以「所屬編號-主題-版本」命名，方便追蹤與搜尋。
  - 如：專案資料以「300專案/231偵查」命名，內部資料以「300-001執行策略」
- 日誌類以 YYYY-MM-DD 命名。
- 版本以 00、01、02，以此類推。
