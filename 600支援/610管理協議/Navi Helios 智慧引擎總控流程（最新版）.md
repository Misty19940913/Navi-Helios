### 1\. 語音 + 指令觸發層（Jarvis 感）

- 喚醒語：Hey Navi
- 實作：Python + Vosk 語音辨識 + n8n webhook
- 功能：語音啟動 Obsidian / 啟動記事 / 啟動 RAG 檢索 / 朗讀回覆

### 2\. 筆記內容理解 / 分類 AI Agent

- 輸入：自動掃描 Inbox + 日記區筆記
- 處理：透過 n8n 傳至 Ollama 模型 → 判斷為靈感 / 專案 / 覺察類型
- 輸出：自動分類至適合資料夾，或生成一份 “行動建議清單”

### 3\. [[RAG+學習型推薦模組]]（Semantic Search）

- 建構：使用 Qdrant / Weaviate + 文本向量化（SentenceTransformer）
- 功能：可查詢所有筆記，並回答「我之前提過XX嗎？」「最近我在哪個領域有行動鬆動？」
- 回答由 Ollama 完成整合說明

## 🔄 指令流程設計

### 1️⃣ 你在 Cursor 下達「複合型指令」

例如：

```markdown
plaintext請整理我過去三週寫的關於『決策疲勞』的內容，撰寫成一篇風格接近《反脆弱》的文章草稿。
```

### 2️⃣ Cursor 將此指令標記並觸發 n8n → /prompt webhook

---

### 3️⃣ n8n 負責以下協作行為：

1. 呼叫 RAG 系統（→ Qdrant）
	- 找出相關片段
2. 使用 Ollama 分析指令 → 精煉提示 / 優化格式 / 檢查誤解
3. **以語意總結＋反問方式回報給你** ，例如：
	> 「共找到3段過去提及『決策疲勞』的文字，擬定以下文章骨架。是否繼續？」

---

### 4️⃣ 你確認後，n8n 將精煉 Prompt 與資料餵給 Cursor 插件 API（或本地編輯插件）

- 在 Cursor 中自動產出文章草稿
- 可直接在該檔案內加注記 / 自我修改 / 插入說明語句

---

### 5️⃣ 成果自動同步至 Obsidian → 放入特定資料夾（如 /040 自媒體工作區/Drafts/）

- 你打開 Obsidian 就能讀取最終成果
- 可啟用提醒：「這份草稿是否要排程到本週發文？」
