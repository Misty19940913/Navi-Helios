### 第 1 階段｜系統環境搭建（基礎平台）

**目標** ：建好基礎運行環境，讓所有模組可以串連

#### 🔧 必要建置：

- 安裝 **Docker Compose**
- 部署本地環境：
	- `n8n` （自動化工作流）
	- `Ollama` （本地大型語言模型）
	- `Qdrant` （RAG語義搜尋用向量資料庫）

#### ✅ 任務完成條件：

- 能啟動 n8n 編輯介面
- 能透過 curl 或 API 與 Ollama 溝通
- Qdrant 啟動並能收錄 dummy 文本向量

---

### 第 2 階段｜智慧分類工作流設計

**目標** ：讓 Obsidian 收件區筆記透過 AI 自動分類並更新 metadata

#### 🧩 建置內容：

- Obsidian → Webhook（透過 Post 插件 or CLI Sync）
- n8n 工作流觸發：
	- 抓取 Markdown → 使用 Ollama 判斷類型（靈感 / 專案 / 日記）
	- 插入 YAML（ `tags` 、 `type` 、 `date` ）
	- 回寫原始筆記
- 判斷為「靈感」時，自動產出行動任務，寫入當日 `010 夢想日記` 中

#### ✅ 任務完成條件：

- 自動化流程可分類並插入 YAML
- 可自動生成任務並 append 至當日工作日誌

---

### 第 3 階段｜語義搜尋與學習型推薦系統（RAG核心）

**目標** ：讓系統能回答「我曾經說過什麼？」並整合過去的思考與資料

#### 📚 建置內容：

- 將 Obsidian 筆記餵入 Ollama Embedding API
- 切片 → 向量化 → 存入 Qdrant
- 設定查詢介面：
	- 使用 Telegram、Web 或 Chat UI 查詢語句
	- 呼叫 Qdrant → 擷取片段 → Ollama 解讀並生成完整回答

#### ✅ 任務完成條件：

- 可從任意筆記搜尋語意相符段落
- 回應能整合歷史思考 / 提出相關行動建議

---

### 第 4 階段｜語音與遠端觸發層（Jarvis模組）

**目標** ：像賈維斯一樣，能用語音或簡訊指令啟動整個智慧流程

#### 🗣️ 建置內容：

- Telegram Bot + n8n webhook 指令：
	- `/process inbox` 、 `/query 最近的反脆弱筆記`
- 日後升級可搭配語音（如：whisper + hotword on Desktop）
- 可選建桌面快速鍵、GUI 啟動器

#### ✅ 任務完成條件：

- 可用 Telegram Bot 操控至少兩種指令
- 能啟動筆記分析或搜尋任務並即時獲得回答

## 🎯 建置建議節奏

1. **第1與第2階段可同時進行** （邊架設邊做分類流程）
2. 第3階段建議 Obsidian 筆記有 50+ 篇以上才開始建立 RAG 資料庫
3. 第4階段為進階拓展，屬於魔王等級主線
