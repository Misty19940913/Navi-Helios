# 財務交易策略 - 開發規範

> **適用範疇**：`300專案/330個人金融知識庫/` 下的所有 `.mq5`、`.pine` 與相關 `.md` 文件。
> **唯一真實來源**：所有交易策略的概念基礎，必須以 `330個人金融知識庫` 中的內容為準，確保不與既有投資哲學衝突。

---

## 程式語言規範

- **MQL5 檔案**：MetaTrader 5 指標與 EA 開發
- **Pine Script 檔案**：TradingView 指標開發
- 兩種語言需**保持邏輯一致性**，便於跨平台驗證

---

## 指標開發規範

- 使用標準技術指標庫，避免重複實現基礎功能
- 指標計算必須包含適當的錯誤處理
- 所有輸入參數必須有**合理的預設值和範圍限制**
- 指標輸出必須清晰標示（上漲/下跌信號）

---

## 風險管理規則（核心紅線）

| 規則 | 參數 |
|---|---|
| 單筆最大回撤 | ≤ **2%** |
| 總帳戶最大回撤 | ≤ **15%** |
| 風險參數管理 | **禁止硬編碼**，必須使用可配置變數 |
| 倉位計算 | 根據帳戶大小動態調整 |

---

## 回測與驗證標準

- **自動生成回測報告**：年化收益、最大回撤、夏普比率、勝率
- **最小勝率要求**：≥ 50%
- **回測期間**：至少包含一個完整市場週期
- **輸出格式**：Excel 或 JSON

---

## 命名規範

| 對象 | 格式 | 範例 |
|---|---|---|
| 函數 | `snake_case` | `calculate_stop_loss()` |
| 常數 | 大寫 | `WINDOW_50 = 50` |
| 策略類 | `PascalCase` + `Strategy` 後綴 | `MaBreakoutStrategy` |
| 變數 | 描述性名稱，**避免縮寫** | `entry_price`, `stop_distance` |

---

## 日誌與追蹤

- **每筆交易記錄欄位**：時間戳、商品、價格、信號、損益
- **輸出格式**：CSV 或 JSON
- **實盤交易**：必須加入 `correlation_id` 做追蹤
- **錯誤日誌**：必須包含詳細錯誤訊息和堆疊追蹤

---

## API 與安全性

- **禁止**在程式碼中硬編碼 API 金鑰
- 使用環境變數或配置檔案管理敏感資訊
- 實作 API 呼叫的錯誤重試機制
- 加入 QPS 限制避免過度請求

---

## 程式碼品質要求

- 所有函數必須有**適當的註解說明**
- 複雜邏輯必須**分解為小函數**
- 避免重複程式碼，使用函數重構
- 保持程式碼的可讀性和可維護性
