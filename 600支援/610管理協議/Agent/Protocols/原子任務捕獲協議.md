# 協議：原子任務捕獲 (Atomic Task Capture)

## 0. 啟動條件 (Triggers)
- 當我說「幫我記下這個任務」、「記錄待辦」或輸入 `//CAPTURE_TASK` 時。
- 當對話中產生明確的下一步行動 (Next Actions) 時。

## 1. 執行程序 (SOP)

### 第一階段：信息提取
1. **任務名稱**：精煉為一句具備動詞的描述（如：重寫自媒體審計協議）。
2. **範疇 (Subject)**：判斷其屬於哪個 MOC 領域。
3. **溯源 (Parent)**：判斷其源自哪個專案或目標。

### 第二階段：結構化生成
自動產出符合 `610-100YAML規範` 的標準欄位：
- `tags`: `[系統/任務]`
- `status`: `todo`
- `time_create`: 今天日期
- `time_end`: 預設為今天（或根據語境推斷）
- `priority`: (根據急迫性判斷)

## 2. 存儲規範

檔案名稱應為：`任務描述.md`
存儲路徑：`200週期紀錄/230任務池/`
