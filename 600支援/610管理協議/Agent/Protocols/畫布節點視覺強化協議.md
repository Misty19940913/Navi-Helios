# 畫布節點視覺強化協議 (Canvas Node Visual Enhancement Protocol)

## 1. 核心定義 (Core Definition)
為了提升 Obsidian Canvas 的可讀性與視覺美感，放棄使用單純的 `file` 類型節點，改為使用 `text` 類型節點作為「主題卡片」。

## 2. 標準格式 (Standard Format)
每個主題卡片必須遵循以下 Markdown 結構：
```markdown
## [[檔案名稱 (顯示名稱)]]
{來自原始筆記的簡要說明/定義}
```

## 3. 視覺規範 (Visual Standards)
- **節點類型**：優先使用 `text` (矩形文本框)。
- **寬高比**：
    - 單行簡介：建議寬度 300-400，高度 140-160。
    - 長描述/多行：高度根據內容增加（如 240-300）。
- **色彩策略**：
    - **技能/策略**：遵循支柱色調 (如 Pillar 2 = 紫色/藍色)。
    - **公式/核心**：使用醒目色 (如 紅色/黃色)。

## 4. 內容提取邏輯 (Content Extraction)
當 AI 被要求在畫布中建立節點時，必須：
1. 讀取目標筆記（原子化知識庫範疇）。
2. 提取筆記中的 `abstract` 或首段「定義 (Definition)」作為摘要。
3. 封裝至上述標準格式中。

## 5. 聯接模式 (Connection Logic)
- **相加項**：連線標籤標註為 `+`。
- **驅動/轉化**：連線標籤標註為行為動詞 (如 `傳遞`、`封裝`)。
- **層級**：使用 `fromSide` 與 `toSide` 確保線條不交錯，呈現清晰的垂直或水平流。
