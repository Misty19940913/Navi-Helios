# 知識原子化與畫布轉化協議 (Legacy)
> [!WARNING] 過時協議 (Deprecated)
> 此文件為舊版協議，現已由 **`.agents/workflows/note-to-canvas.md`** 正式取代。
> 執行相關任務時，請優先參考新版工作流，並遵循 `.agents/skills/atomize-note` 中的優化邏輯（如正文原地升級）。

## 1. 核心目標
定義一套將「長篇/舊版戰略文件」解析並轉化為「畫布視覺節點」的具體流程，實現知識從線性文字到網狀視覺的過渡。

## 2. 三階段轉換流程

### 第一階段：標籤化與原子化 (Atomization)
- **辨識概念**：從原始文檔中提取具備「獨立解釋價值」的名詞（如：技能堆疊、個人壟斷）。
- **建立實體**：
    - 若該知識點尚未存在於 `410原子化知識庫/`，則建立新檔。
    - 若已存在，則根據新資料補充內容（重點提取定義與機制）。
- **YAML 規範**：確保具備 `parent` (溯源) 與 `subject` (範疇) 屬性。

### 第二階段：拓撲定位與去中心化 (Topology)
- **支柱歸位**：判斷知識點所屬的系統支柱（Pillar 1: 核心、Pillar 2: 能力、Pillar 3: 價值包裝、Pillar 4: 槓桿、Pillar 5: 定位）。
- **多重實例 (Shadow Nodes)**：
    - **規則**：若一個知識點同時連結多個不同區域，不允許跨越整個畫布連長線。
    - **動作**：在目標區域建立該節點的「影子實例」(Shadow Instance)，保持連線簡潔且具備局部上下文。

### 第三階段：視覺強化封裝 (Visual Packaging)
- **實施標準**：遵循 `畫布節點視覺強化協議`。
- **節點轉換**：將 `file` 類型改為 `text` 類型。
- **內容組合**：標題使用 `[[檔案名稱]]` + 提取筆記中的 `abstract` 或 `定義 (Definition)` 段落。
- **視覺校準**：統一調整寬高 (寬 350-400)，並賦予對應支柱的顏色編碼。

## 3. 實戰回溯範例 (以 232-150 為例)
1. **提取**：從 `232-150` 識別出「智能模仿」。
2. **原子化**：更新 `智能模仿.md`，提取其「解構、抽象、重組」機制。
3. **佈置**：將其置於支柱 3 (價值包裝) 與支柱 5 (定位) 的交界。
4. **影子化**：在「創造力循環」中建立該節點的影子實例以維持環狀邏輯。
5. **視覺化**：轉化為文本卡片，顯示其核心定義。

## 4. 觸發條件
- 當使用者要求「整理畫布」或「更新邏輯架構」時。
- 當系統偵測到長篇戰略文件已完成階段性任務，需進入「系統化存儲」時。
