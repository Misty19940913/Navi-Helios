# 協議：自媒體審計 (Content Audit Protocol)

## 0. 啟動條件 (Triggers)
- 當使用者輸入 `//audit` 或要求「審核文案」時。
- 當完成初稿，準備進入發布階段前。

## 1. 執行程序 (SOP)

### 第一阶段：禁令掃描 (Forbidden Scan)
首先檢查文中是否踩到以下紅線：
- **結論式結尾**：出現「總之」、「讓我們一起」、「預祝大家」等句式。
- **全包式敘述**：試圖一次講完所有道理，缺乏單點穿透力。
- **廉價安慰**：使用雞湯、加油、建議你等「平庸視角」的安慰。
- **遺失主權**：語氣過於溫和或在尋求認同。

### 第二階段：四維度量化評分 (Scoring Model)
對標 `Misty` 的核心形象，給予 0-10 分的客觀評價：

1. **文字脫水度 (Dehydration)**：
   - 廢話、虛詞、重複形容詞的清理程度。
   - 節奏：是否符合「短句、高頻、冷峻」？
2. **狩獵感/穿透力 (Penetration)**：
   - 是否直視殘酷真相？
   - 是否具備「俯視風險」的主權感？
3. **場景建構感 (Atmosphere)**：
   - 是否融入「刑警/交易者」等具象場景？
   - 讀者能否感受到環境的冷靜秩序？
4. **身份轉化效率 (Conversion)**：
   - 最後的 CTA 是否具備挑釁感與選擇主權？

### 第三階段：邏輯缺陷診斷
- 指出內容在哪裡向「平庸」妥妥、在哪裡顯得軟弱。
- 定位 3 個可能的「滑走點」（讀者會離開的位置）。

## 2. 輸出範本 (Output Template)

```markdown
## 🔍 自媒體文章稽核報告

| 維度 | 評分 | 診斷詳述 |
|---|---|---|
| **脫水度** | /10 | |
| **穿透力** | /10 | |
| **場景感** | /10 | |
| **轉化率** | /10 | |

### 🚩 違禁點偵測
- (列出偵測到的結論式結尾或平庸語句)

### 🩺 滑走點診斷
1. (位置+理由)
2. (位置+理由)

### 💡 建議啟動
- [[敘事極致化重構協議]] (若評分低於 7 分)
```

---
**僅產出稽核報告，不進行重寫，除非使用者要求進一步執行重構協議。**
