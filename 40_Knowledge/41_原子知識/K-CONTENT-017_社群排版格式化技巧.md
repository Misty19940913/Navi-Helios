# 社群平台排版格式化技巧

## 核心主張

**格式決定閱讀體驗，相同的內容不同格式效果差異可達數倍。**

## 符號選擇

### 列表符号：• 而非 -

```
錯誤示範：
- 第一點
- 第二點
- 第三點

正確示範：
• 第一點
• 第二點
• 第三點
```

為何：真正的實心點（•）視覺上更清晰，在手機窄屏上比Dash更美觀。

## Twitter/推文寫作格式化

遵循此結構：

```
1. 開場鈎子（吸引眼球）
2. 核心內容（1-2個要點）
3. 總結句（濃縮精華）
4. 呼籲行動（ex: 你有什麼看法？）
```

## LinkedIn 特殊注意

- **頭3句話**：決定是否展開全文（演算法邏輯）
- 不要在開頭用「這篇文章要談...」之類廢話
- 直接開門見山，用數據或強烈觀點開場

## 行距與留白

- 段落間距：至少一行空行
- 不要一個段落太長（最多5句，參考1-3-1法）
- 手機閱讀舒適性優先

## 常見錯誤

| 錯誤 | 後果 |
|------|------|
| 用數字列表而非 • | 視覺擁擠 |
| 段落太長 | 讀者視覺疲勞直接跳出 |
| 全部大寫或過多驚嘆號 | 顯得垃圾郵件化 |
| 沒有呼籲行動 | 讀者看完沒反應，不知道要幹嘛 |

## 底層原理

- 網絡閱讀是「掃描式」— 格式幫讀者快速定位
- 視覺空白 = 認知空白，讓大腦有喘息空間
- 呼籲行動（CTA）降低讀者下一步的行動成本
