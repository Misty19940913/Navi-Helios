# PKM×PM 雙軌制

> [!abstract]+ 核心定義
> HQ（處理中心）的兩種加工模式：**PKM（Personal Knowledge Management）** 負責知識的發散與收斂，將外界資訊沉淀為 Areas 的智慧；**PM（Project Management）** 負責行動的追蹤與執行，確保 Projects 在期限內達成產出。

---

## 雙軌對比

| 維度 | PKM（知識管理） | PM（專案管理） |
|------|----------------|----------------|
| **核心** | 知識的發散與收斂 | 行動的跟蹤與執行 |
| **輸入** | 外部資訊（書籍、課程、文章） | 任務、截止日期、交付物 |
| **邏輯** | Resources → Areas（智慧沉淀） | Projects → 產出（價值兌現） |
| **目標** | 成為更好的人 | 完成特定專案 |
| **時間視角** | 長期主義 | 短期/中期主義 |
| **衡量標準** | 認知深度與洞察廣度 | 交付率與完成度 |

---

## PKM 軌：知識沉澱流程

```
外界資訊 → Resources → 內化萃取 → Areas（領域智慧）
```

**範例**：
- 讀完《原子習慣》→ 萃取習慣養成原則 → 加入「自我成長」Area
- 研究競爭對手 → 萃取市場洞察 → 加入「品牌經營」Area

**關鍵問題**：*這份資訊跟我目前關注的哪個領域有關？*

---

## PM 軌：專案執行流程

```
目標 → 拆解 → Projects → 任務追蹤 → 產出
```

**範例**：
- 「推出線上課程」→ 拆解為「建課程網站」「錄製影片」「文案撰寫」→ 追蹤進度 → 交付上線

**關鍵問題**：*這個專案的下一個行動是什麼？截止日期是什麼？*

---

## 為什麼需要雙軌？

| 只有 PKM | 只有 PM |
|----------|---------|
| 輸入很多但沒有產出 | 忙於執行但認知停在原地 |
| 變成「資料囤積」 | 缺乏深度积累 |
| 感覺很充實但沒有成長 | 忙綠但不知道為何而忙 |

**雙軌並行** = 執行力（PM） + 認知力（PKM）同步推進，防止「忙而無成長」或「學而無產出」。

---

## 與 Life OS 的關係

[[Life-OS-MOC]] 現有系統中：
- **PM 軌已有完整建模**：Goal → Project → Task 三層執行串聯
- **PKM 軌相對薄弱**：缺乏「知識如何沉淀為 Area 成長」的明確定義和流程

**實作建議**：在 Life OS 中加入「知識內化 Ritual」，例如：
- 每週從 Resources 提煉至少一筆記錄到對應 Area
- 記錄時用自己的話重述，而非單純剪藏
