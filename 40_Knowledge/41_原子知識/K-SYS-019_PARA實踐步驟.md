# PARA 實踐步驟

## 核心定義

面對多年積累的數位垃圾場，試圖逐一分類只會導致挫敗。你需要的是一場「資訊破產宣告」，給自己一個重生的機會。

## 核心概念

| 概念 | 說明 |
|------|------|
| 資訊破產（Information Bankruptcy） | 將所有舊資料直接移入封存資料夾，從零開始建立新系統 |
| 及時組織（Just-in-Time） | 只有在任務真正啟動時才建立分類，拒絕預建空資料夾 |
| 週五重設（Friday Reset） | 每週例行維護儀式——清空 Inbox、更新專案狀態、將已完成的專案推入存檔 |

## 精確的廚房組織類比

| 廚房物品 | PARA 類別 |
|----------|-----------|
| 鹽巴與胡椒（每天用） | Projects / Areas |
| 日常餐具 | Areas |
| 特殊香料 | Resources |
| 火雞烤盤（每年用一次） | Archives |

## 決策樹

1. 這是否與我目前有截止日期的具體目標相關？→ Projects
2. 否則，這是否與我需要長期維護品質的責任相關？→ Areas
3. 否則，這是否是我純粹感興趣、未來可能有用的主題？→ Resources
4. 否則，這是否已失去時效性但未來可能想參考？→ Archives
5. 否則 → 直接刪除

## 行動指南

1. 不預先建立：沒有內容，就不要建立空資料夾
2. 不追求完美：分類本身不是目的，完成專案才是
3. 不讓過期資訊留在視覺中心：專案結束，立即移入封存
4. 選擇性週回顧：每週只針對每個類別中「最久沒碰」的 4 則筆記進行回顧

> [!abstract] 核心洞見
> PARA 的失敗通常是「過度設計」而非「不夠整潔」——系統是為了服務生活，而非成為生活的負擔。
