# PKOS 核心功能模組 — 構建「從數據到智慧」的四層轉化體系

> [!abstract] 核心洞見 (Insight)
> **功能不再是孤立的點，而是價值產出的「流水線」。** PKOS 核心功能模組的設計哲學是：將 **「主權數據 (Local Files)」** 與 **「異質推理 (Multi-Model AI)」** 深度耦合。系統不應只是一個存放文字的倉庫，而應是一個能針對特定任務自動配置專家（Agents）的 **「動態實驗室」**。透過四大模組的協同，我們實現了從「資訊存儲 (Base)」到「專家執行 (Agent)」的完整進化。這將極大化超級個體的認知槓桿，使你一人即能完成一個團隊的工作量。

### 定義與機制 (What & How)

#### 四大核心功能模組 (The 4 Pillars of Capability)

1. **基礎層：筆記系統 (Vault Base) — [數位領土]**
    - **特徵**：本地 Markdown 儲存、雙向連結、SSOT 原則。
    - **價值**：提供系統的 **「長期記憶」** 與 **「特定知識存量」**。它是 AI 所有推理活動的物理根源。
2. **交互層：AI Cursor 模式 (Direct Interaction) — [直覺操作]**
    - **特徵**：選取文字即調用、上下文感知的 Patch 修改。
    - **價值**：消除 **「操作摩擦」**。讓 AI 介入文字的「原子級」修改，實現「選中即重構，呼叫即行動」。
3. **智能層：多模型 API 路由 (Model Routing) — [大腦中樞]**
    - **特徵**：自動根據任務複雜度切換模型（如 Gemini 1.5 Pro 讀長文，Claude 3.5 Sonnet 寫邏輯）。
    - **價值**：**「推理成本優化」**。確保每個任務都使用最強（或最經濟）的「認知算力」。
4. **應用層：AI Agent 系統 (Expert Agents) — [角色演化]**
    - **特徵**：預設專業角色（如：思考教練、盲點偵測器、內容編輯）。
    - **價值**：提供 **「異質視角」**。AI 不再是通才，而是能針對具體專案進行深度優化的專業代理。

#### 運作邏輯：從資訊到行動的級聯
- **基礎資料 (Vault)** -> **意圖捕捉 (Cursor)** -> **算力分配 (Routing)** -> **專家執行 (Agent)**。

### 實踐與案例 (Action & Proof)
- **真實案例**：Navi 處理「職涯迷茫」的知識生產線。
  - **步驟 1 (Interaction)**：Navi 選取一段混亂的當下感受，透過 **AI Cursor** 進行初步分類，識別出 3 個核心痛點。
  - **步驟 2 (Agent - 策略)**：呼叫 **「策略顧問 Agent」**，調取 L3 等級的職業規劃原則（K-CAREER-002），生成 3 個具備可行性的「奧德賽路徑」。
  - **步驟 3 (Agent - 批判)**：呼叫 **「盲點偵測器 Agent」**，專門針對上述路徑進行「壓力測試」，指出財務風險點。
  - **步驟 4 (Storage)**：最終決策被自動封裝為一份標準化的原子筆記，存入 `/20-Project/`。
  - **成果**：Navi 在 15 分鐘內完成了原本需要一週反覆糾結的決策過程。他明白：**「我不是在跟機器聊天，我是在指揮一個專業顧問團。」**
- **行動指南**：
  1. **建立「角色清單」**：為你的 PKOS 定義 3 個最常用的 Agent 角色，並為其編寫詳細的 System Prompt。
  2. **優化「模型路由」**：設定自動化規則，例如「超過 5000 字的輸入一律導向 Gemini 1.5 Pro」。
