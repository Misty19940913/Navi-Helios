# PARA 四大分類法

> [!abstract]+ 核心定義
> PARA 是「正在流行」資訊的**流動狀態**分類法，而非靜態資料夾。每個分類代表資訊在系統中的活躍程度與使用目的：**P = 正在燃燒 / A = 持續灌溉 / R = 感興趣圖書館 / Arch = 冷卻記錄**。

---

## 四分類詳解

| 分類 | 全名 | 隱喻 | 活躍程度 | 截止日期 |
|------|------|------|----------|----------|
| **P** | Projects（專案） | 正在燃燒的事 | 🔴 高 | 有明確截止日 |
| **A** | Areas（領域） | 需要持續灌溉的地 | 🟡 中 | 無截止但需持續投入 |
| **R** | Resources（資源） | 感興趣的圖書館 | 🔵 低 | 無明確時限 |
| **Arch** | Archives（歸檔） | 完成或冷卻的紀錄 | ⚪ 休眠 | 不再活躍 |

### P — Projects（專案）

> 「正在燃燒的事」

- **定義**：有明確目標與截止日期的行動集合
- **範例**：2 週內完成報告、3 個月內推出產品原型
- **特徵**：時限壓力驅動執行，不可長期搁置

### A — Areas（領域）

> 「需要持續灌溉的地」

- **定義**：長期責任領域，無截止日期但需持續投入
- **範例**：品牌經營、健康管理、財務規劃
- **特徵**：維護保養性質，不能間斷太久

### R — Resources（資源）

> 「感興趣的圖書館」

- **定義**：當前的興趣主題或未來可能用到的參考資料
- **範例**：研究 AI 趨勢、收藏的設計參考、潛在機會研究
- **特徵**：處於「還不確定怎麼用」的階段

### Arch — Archives（歸檔）

> 「完成或冷卻的紀錄」

- **定義**：不再活躍但值得保留的歷史資料
- **範例**：已完成專案的文件、已放棄的想法、過時的市場報告
- **特徵**：保持系統清爽的關鍵，壓縮資訊密度

---

## PARA 的核心原則

> **資訊是流動的，不是靜態存放的。**

同一份資料在不同時間點可能流經所有四個狀態：
```
Resources → (開始研究) → Areas → (啟動專案) → Projects → (完成) → Archives
```

## 實踐要點

- **進 Resources 的標準**：還不確定跟哪個 P 或 A 有關，但值得保留
- **進 Projects 的標準**：有明確時限和交付物的行動
- **進 Areas 的標準**：長期責任，需要定期投入（如每週運動）
- **進 Archives 的標準**：不再需要時常查阅的已完成資料

> [!TIP]
> 如果一個資料夾讓你維護起來很痛苦，簡化它。系統是活的。
