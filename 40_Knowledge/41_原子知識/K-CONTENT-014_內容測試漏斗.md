# 內容測試漏斗：短文先行，長文後進

## 核心主張

**花大把時間寫長文前，先用短文測試市場反應。**

不浪費時間在沒人看的題目上。

## Lean Writing 概念

來自「精益創業」（Lean Startup）的類比：

```
構思 → 短文發布 → 觀察數據 → 決定是否寫長文
```

## 為何先寫短文

| 問題 | 傳統做法 | Lean Writing |
|------|----------|--------------|
| 花時間寫長文但沒人看 | 常見且代價高 | 先用短文試水溫 |
| 不知道讀者對什麼有興趣 | 猜測/直覺 | 看互動數據 |
| 題目方向不確定 | 選一個悶頭寫 | 先多元化測試 |

## 有效信號（如何判斷短文成功）

**黃金標準**：有陌生讀者發DM詢問詳細內容

> 「就算沒有按讚和分享，只要有人DM你問內容，這篇文章極有價值。」

其他信號（強度遞減）：
- 留言詢問
- 私訊回覆
- 轉發/分享
- 按讚

## 實作節奏

1. **初期（1-4週）**：随意写任何感兴趣题目，大量化短文收集数据
2. **中期（信號出現後）**：专注获最多互动的1-2个主题
3. **長期**：圍繞確立的主題寫深度長文/產品

## 常見錯誤

- **太長的內容**：短文太短變空洞 → 要足夠具體讓人有收穫
- **過早投入**：還沒驗證題目就寫長文 → 浪費時間
- **只看按讚數**：按讚不等於價值，私訊詢問才是真正信號
