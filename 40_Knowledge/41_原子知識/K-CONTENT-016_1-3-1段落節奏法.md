# 1-3-1 段落節奏法

## 核心主張

**每個段落用1-3-1節奏寫，讀者會被自動引導持續閱讀。**

句式有節奏 → 閱讀流暢 → 讀者不停

## 結構

每個段落內：

| 部分 | 句數 | 功能 |
|------|------|------|
| 開頭句（1） | 1句 | 立場/主題句，強烈有力 |
| 內文（3）| 3句 | 支撐、例子、延伸 |
| 結尾句（1） | 1句 | 回扣主題或轉接 |

## 為何有效

- 讀者的眼睛有「休息點」— 1-3-1 是自然呼吸節奏
- 每個段落足夠短（5句）→ 手機時代極易消化
- 結構預期 → 降低認知負擔

## 視覺化

```
[立場句] ← 1句，強烈有力
[支撐句]
[支撐句] ← 3句，具體細節
[支撐句]
[回扣/轉接句] ← 1句，總結或引導
```

## 實際例子

> **網站加載速度決定留存率**（1）
> 當頁面加載超過3秒，53%的用戶會離開。（3）
> Google的研究顯示，每增加1秒等待，轉化率下降20%。
> 亞馬遜也曾透露，頁面速度每落後100毫秒就影響1%營收。
> 這就是為什麼性能優化不是可選項，而是基本功。（1）

## 對比：壞段落的問題

- **太長**：一段10+句 → 讀者視覺疲勞放棄
- **全是抽象概念**：沒有具體支撐 → 論點飄在空中
- **沒有節奏感**：句式長短不均 → 閱讀體驗差

## 適用場景

- 所有社群平台的正文寫作
- Newsletter
- 部落格文章
- LinkedIn長文
