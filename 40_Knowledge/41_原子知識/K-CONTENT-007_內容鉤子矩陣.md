# 內容鉤子矩陣

> [!abstract] 核心洞見 (Insight)
> 內容鉤子矩陣

### 定義與機制 (What & How)
## 核心定義

內容鉤子（Hook）是將目標讀者的「痛點/渴望」轉化為高情緒喚起的內容開場技術，透過觸發特定心理弱点或願景，瞬間抓住注意力並引發繼續閱讀的動機。

## 四大鉤子類型

| 類型 | 核心情緒 | 範例格式 |
|------|---------|---------|
| **痛點鉤子** | 恐懼/焦慮 | 「為什麼你明明看對方向，卻在起漲前被掃出場？」 |
| **願景鉤子** | 渴望/希望 | 「2026 年，一個人的公司可以做到多大？」 |
| **共鳴鉤子** | 識別/理解 | 「月薪 5 萬，負債 600 萬，我是如何活下來的？」 |
| **知識鉤子** | 好奇/認知 | 「強迫症的極致：如何用一個筆記軟體管理整個人生？」 |

## 東黎苑鉤子庫（實戰範例）

### 場景痛點鉤子
- `AI 詐騙`：AI 深偽詐騙：剛才這通電話，連你媽的聲音都能造假。
- `資本黑箱`：為什麼你的錢進了虛擬錢包就找不回來？
- `債務焦慮`：負債者的強迫症：為什麼我連買咖啡都要紀錄？

### 高頻情緒詞彙（哨兵視角）
- `累積性創傷`：日常無感耗損，就像管線微小漏水
- `他人的悲劇組成的日常`：將工作或現實痛苦視為常態的麻木感
- `系統漏洞`：為什麼月入六位數依然存不到錢、年終依然是負的？

## 鉤子×內容框架整合

Hook 嵌入位置：

```
1. [鉤子 Hook] → 觸發痛點/渴望，0.3 秒內抓住注意力
2. [問題定義 Problem] → 放大問題細節，建立「這就是我」的識別
3. [核心方案 Solution] → 提供反直覺或深層邏輯，重構問題
4. [行動步驟 Action] → 給予具體 3 步驟，降低執行門檻
5. [召喚行動 CTA] → 微任務導向，引導留言/私訊/轉發
```

## 與其他框架的關係

- **上游**：SWOT×WHO×WHAT×HOW 確認品牌定位語境
- **核心**：鉤子必須與品牌的核心價值主張一致，非純標題黨
- **下游**：鉤子 → PSA 框架 → 內容矩陣 → 平台發布策略

## 使用時機

- 每次創作前先確認「這篇的鉤子是什麼」
- 每週排程時檢視鉤子多樣性（避免同一類型疲勞）
- A/B 測試不同鉤子版本，收集回饋迭代
