# CODE 知識管理框架

## 核心定義

大腦是用來產生想法的，不是用來持有想法的。CODE 框架將第二大腦變成你的「認知卸載」工具，讓生理大腦回歸思考與創造。

## CODE 四階段

| 階段 | 全寫 | 說明 |
|------|------|------|
| C | Capture（採集） | 只留住與靈魂共鳴的資訊 |
| O | Organize（組織） | 以 PARA 系統分類 |
| D | Distill（提煉） | 找出可掃視的金礦 |
| E | Express（表達） | 利用中間封包實現創意產出 |

## 採集判斷標準

| 標準 | 問題 |
|------|------|
| 啟發性 | 它是否觸發了你的靈感或情緒？ |
| 有用性 | 它能否應用在目前的專案中？ |
| 個人化 | 它是你獨特觀察、回憶或反思的紀錄嗎？ |
| 令人驚訝 | 它是否挑戰了你既有的思維模型？ |

## 提煉的「20% 規則」

每一層級的內容應僅約為前一層級的 20%。漸進式提煉：100% → 25% → 5% → 1%。

## 中間封包（Intermediate Packets）

將過去成果拆解成的可重複調用模組，包含：
- 提煉過的筆記（過去讀過的摘要）
- 失敗的素材（未被採用的草稿）
- 進行中的半成品
- 最終交付成果
- 他人的文件

## 「電力法則」

通常只有不到 1% 的筆記會達到「重新組合」層次——這是正常的。

## 行動指南

1. 宣告「資訊破產」：將現有雜亂檔案移入標註日期的封存資料夾，從零開始
2. 每次「觸碰」筆記時，順手讓它變得比之前更容易閱讀一點
3. 從「被動接收者」進化為「數位創作者」

> [!abstract] 核心洞見
> CODE 的本質是將「知識管理」從「倉儲」變成「投資」——知識的價值在於持有如何在未來正向影響他人與自己。
